# Information-Limitation Assessment: what the cross-track source wall means for the modelling outcome

Date: 2026-06-24

This note asserts no new evidence and changes no confidence score. It synthesises a
pattern that now recurs across every active validation front and states, plainly,
what that pattern implies for the programme's empirical standing.

## The observation

Three independent validation domains are blocked at the same step:

| Domain | Modelling side | What is openly available | What is missing (the wall) |
|---|---|---|---|
| Mineral refractive index | structural RI model form predeclared; descriptors frozen | excellent polytype-specific CIFs (COD: WO1T, Wollastonite-2M, etc.) | measured optical constants + density **locked to the same single polytype**; literature gives only broad pooled handbook ranges |
| Surface-roughness PSD | two comparators built + synthetically validated | scatter-derived PSD parameters (e.g. Sulc 2023 Table 3); structural metadata | the **independent topography PSD as numerical rows**; published only as figures, or "on request", or paywalled |
| Peroxide torsion | crowding descriptors predeclared | qualitative/ratiometric routes; some single aggregate barriers | a **numerically-exposed 1-D torsional profile** for a fresh, identity-locked target |

The failure mode is identical: the programme can secure one half of each required pair
(usually the structural / parametric half) openly and numerically, but the other half —
the independent measured observable, locked to the same identity, in extractable numerical
form — is systematically not published that way.

## Why the wall has this exact shape

The validation discipline demands, for one identity (material / polytype / molecule):
(a) a structural or descriptor source, and (b) an independent measured-property source,
both numerically tabulated, both locked to that same identity, with the model frozen before
the property is seen. Published literature violates this in predictable ways:

- It pairs **loosely**: broad ranges spanning multiple polytypes; properties quoted without
  the structure that was actually measured.
- It publishes the **cheap half openly** (CIFs, fitted spline parameters) and the
  **expensive half** (measured optical constants, raw AFM PSD curves, full torsional PES)
  as figures, on request, or behind paywalls.
- It leaves **identity implicit** (which polytype / conformer / specimen was measured).

This is not bad luck in three searches; it is how the experimental literature in these
fields is habitually formatted. More searching does not dissolve it.

## What it means for the modelling outcome

1. **The binding constraint is data acquisition, not modelling.** The comparators and the
   RI model form are built and self-consistent. Additional model variants have near-zero
   marginal value until identity-locked held-out data is secured. Effort should move from
   "more model" to "more data".

2. **The confidence cap is structural, not transitional.** Inferential convergence sits at
   5.6, cross-domain equivalence 5.2, unification-thesis support 4.8, capped at 6.25 pending
   a measured material-property prediction that passes without endpoint fitting. Because every
   domain is blocked on the same data wall, this cap will **not** move through literature
   search alone. It should be described as a structural feature of the current evidence
   environment, not a temporary state awaiting one more screen.

3. **The central empirical claim is still untested.** The thesis — a shared topology grammar
   across molecular, material, optical, and interface orderings — has not faced a fresh
   held-out material-property test. Synthetic known-answer validation shows the machinery is
   internally consistent; internal consistency is not external validation. The honest current
   outcome is: a coherent, rigorously-guarded framework whose key falsifiable test has not run,
   because the test data is inaccessible in the required form.

4. **The discipline is the asset.** The programme's trustworthiness is exactly its refusal to
   convert loose or inaccessible data into false positives — no broad pooled rows, no
   figure-digitised curves passed off as locked data, no target-implied coefficients. The
   catalogue of "source-blocked for these specific reasons" is itself a defensible result and
   a publishable methodological contribution.

5. **The deferral risk must be named.** "Source-blocked, defer" cannot become permanent
   epistemic cover. The discipline has to cut both ways: it should prevent false positives AND
   prevent indefinite hiding behind "not yet testable". This requires an explicit stopping
   rule (below).

## A stopping rule, so the discipline stays honest both ways

For each domain: run one bounded, adequately-resourced data-acquisition attempt (author
request, supplementary-data mining, institutional/paid database, or generating the missing
half from deposited raw data — e.g. computing a PSD from a published raw AFM map). If that
attempt does not yield an identity-locked held-out pair, declare the domain **untestable with
available data** and record the consequence plainly: the unification claim remains
**unconfirmed** in that domain, and the limitation is **evidential, not provisional**.

## Reframing near-term success

- Treat data acquisition as the primary work item, not a precondition handled in passing.
- Deliberately add at least one validation domain chosen for **data accessibility** — a field
  that publishes an openly-tabulated, identity-locked held-out benchmark — so the grammar can
  actually face falsification somewhere. Three domains failing the same accessibility test
  hints that domain choice has been driven by where the grammar *should* show, not by where
  testable data *exists*; securing even one domain that satisfies both is the make-or-break
  step for the programme's empirical status.
- Continue building only the machinery that is genuinely unblocked (e.g. predeclaring the RI
  predictor equation + calibration protocol, which must be frozen before a target is seen
  anyway) — but log it as readiness, never as movement toward confidence.

## One-line summary

The programme is not modelling-limited or effort-limited; it is **held-out-data-limited**, and
because that limit recurs identically across domains, the unification thesis should be reported
as coherent-but-unvalidated, with the confidence cap treated as structural until an
identity-locked held-out dataset is acquired in at least one domain.
