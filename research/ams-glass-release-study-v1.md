# AMS Glass Release Study v1

## Purpose

This document deepens the strongest current exit-side anchor case:

- `glass -> air`

The reason this case matters is that it isolates a specific claim already emerging in the programme:

- a whole can transmit well in bulk and still release poorly at its terminal boundary

Glass is the best current benchmark for that claim because the bulk whole already looks reasonably coherent in AMS terms:

- real whole
- distributed permissive transmissive order
- strong enough return-path support
- high slip tolerance

Yet the exit event still appears fragile.

So the central question is:

- why can good bulk glass transmission still end in weak, diffuse, or trapped release?

## Main Claim

Glass release is not limited mainly by bulk transmissive identity.

It is limited mainly by:

- terminal-layer geometry
- local continuity of release pathways
- the ability of the final boundary to convert distributed internal propagation into a viable outward event

That means the release problem sits at the last layer, not across the whole bulk.

## Why Glass Is a Distinctive Release Case

Glass differs from crystal in a crucial way:

- glass carries propagation by distributed permissive reseating
- crystal carries propagation by more patterned nested admissibility

That difference matters on exit.

In glass, the event arriving at the final boundary is:

- coherent enough to propagate
- but not already tightly ordered in a single strongly patterned way

So the release problem is less about relaxing excessive order and more about:

- preventing the final layer from diffusing, fragmenting, or stalling the event

This is why glass is the best exit anchor.

## Three Main Glass Exit Modes

## Mode G1: Clean Release

### Description

The internal event reaches the boundary and leaves the glass in a sufficiently coherent outward form.

### Expected Variable Pattern

- `B_out`: high
- `J_bound`: medium
- `J_trans`: medium/high
- `Q_coh`: medium/high
- `D`: medium

### Interpretation

This is the best-case glass release event.

The glass does not need unusually high boundary-face structuring. What it needs is:

- a terminal layer that does not break the continuity of distributed permissive propagation
- enough through-layer continuity to let the event emerge outward rather than diffuse locally

### What Makes It Work

- surface not overly rough or contaminated
- no severe terminal defect cluster
- no local geometry that forces the event to spread laterally before exit

## Mode G2: Diffuse Release

### Description

The internal event reaches the final boundary, but the terminal layer loosens or spreads it too much before clean outward release.

### Expected Variable Pattern

- `B_out`: medium
- `J_bound`: medium
- `J_trans`: low/medium
- `Q_coh`: medium
- `D`: medium/high

### Interpretation

This is the most likely ordinary failure mode for glass.

The event is not completely lost. It still releases outward to some degree. But the release becomes:

- spread
- less coherent
- less cleanly directional

This fits the idea that the glass whole remains transmissive while the terminal layer degrades the final event more than the bulk degrades propagation.

### What Makes It Happen

- terminal roughness
- local micro-irregularity
- shallow pathway crowding near the surface
- non-catastrophic but meaningful defect burden

## Mode G3: Trapped Release

### Description

The internal event reaches the terminal region but does not complete outward release. Instead it remains locally retained, repeatedly reseated, or shallowly redirected near the boundary.

### Expected Variable Pattern

- `B_out`: low/medium
- `J_bound`: medium/high
- `J_trans`: low/medium
- `Q_coh`: medium
- `D`: medium/high

### Interpretation

This is the strongest release-side frontier within the glass case.

It differs from diffuse release because:

- the event does not simply loosen outward
- it remains locally bounded enough to avoid immediate scatter

It differs from return because:

- the event is not cleanly sent back inward as a structured reflection-like mode

Instead, the event stalls in a shallow boundary-retention region.

### What Makes It Happen

- terminal geometry that permits local reseating but not full release
- weak through-layer continuity in the final release zone
- surface-layer mismatch that is not chaotic enough for immediate scattering

## Why the Terminal Layer Matters More Than the Bulk

The current best reading is:

- the glass bulk remains good enough at carrying distributed permissive propagation
- the terminal layer decides whether that propagation:
  - leaves cleanly
  - spreads
  - stalls

This means the main control point is not the whole glass body, but the final release zone.

That is a strong result because it shows the programme is no longer treating:

- transmission
- release

as the same process.

## Distinguishing the Three Glass Release Modes

## Clean Release vs Diffuse Release

Main difference:

- `J_trans`

In clean release:

- transition continuity survives well enough for outward release to remain coherent

In diffuse release:

- transition continuity weakens enough that the event spreads before or during release

## Diffuse Release vs Trapped Release

Main difference:

- whether the event still finds outward continuation

In diffuse release:

- outward continuation still happens, but badly

In trapped release:

- outward continuation stalls and local reseating dominates

## Trapped Release vs Returned Exit

Main difference:

- returned exit preserves enough organised boundary response to redirect inward
- trapped release remains locally bounded but does not become a clear inward-return event

This distinction remains important, because the release-side model should not collapse all non-release into one class.

## What This Case Suggests About `B_out`

The glass study sharpens the meaning of `B_out`.

Current best definition:

- `B_out` is the degree to which a whole’s terminal boundary can convert already-propagating internal structure into a viable outward event rather than local spread, retention, or return

That is stronger and more specific than earlier shorthand.

## What This Case Suggests About `J_bound` and `J_trans`

Glass exit supports the current split strongly.

### `J_bound`

This appears to control:

- whether the final boundary forms a workable release event at all

### `J_trans`

This appears to control:

- whether the release event can continue through the terminal layer without stalling or broadening too much

That is why glass exit is such a useful benchmark:

- it makes the terminal continuity burden visible without requiring strong crystal-style ordering
