# AMS Exit Pairings Scoreboard v1

## Purpose

This document ranks the current exit-side cases after:

- `ams-exit-geometry-study-v1.md`

The aim is to decide which release-side frontier should be deepened next.

The current exit-side field includes:

- `glass -> air`
- `crystal -> air`
- retained exit modes
- returned exit modes

This scoreboard is meant to answer:

- which cases are already strong enough to act as anchors
- which cases are still frontier cases
- which release-side problem is most worth the next modelling pass

## Scoring Criteria

Each case is scored `1-10` on:

- `Explanatory strength`
  - how coherent the current AMS account is
- `Geometry maturity`
  - how explicitly the case is now modelled geometrically
- `Variable clarity`
  - how cleanly the current variables separate the case from neighbouring outcomes
- `Next-step usefulness`
  - how much the case would improve the programme if deepened next

These are steering scores, not truth scores.

## Ranking by Immediate Value

## Tier 1: Highest Immediate Value

### 1. Glass -> Air

Why it ranks first:

- it is the strongest practical release benchmark
- it is close enough to ordinary transmissive behaviour to remain anchored in the wider programme
- it directly pressures the claim that bulk transmission and release are different tasks
- it exposes surface diffusion and release fragility cleanly

This is the best next case if the goal is to sharpen exit without becoming too abstract too quickly.

### 2. Retained Exit / Boundary Trapping

Why it ranks highly:

- it is the strongest unresolved exit mode
- it forces the model to distinguish failed release from both scattering and reflection
- it pressures the transition-layer and boundary-retention ideas already emerging elsewhere

This is not the cleanest anchor case, but it may be the most fertile failure mode.

## Tier 2: Strong but Slightly Less Urgent

### 3. Crystal -> Air

Why it remains important:

- it is the best ordered release case
- it tests whether a strongly patterned internal event can be converted into viable outward release
- it strengthens `B_out` and `Q_coh`

Why it ranks slightly below glass -> air:

- it is more selective and specialised
- it is slightly less useful as a broad release anchor

### 4. Diffuse Release / Surface Spreading

Why it is useful:

- it gives the best current account of how glass-like release can degrade without collapsing into total incoherence
- it is a realistic release-side failure mode

Why it ranks below trapping:

- it is already partially legible
- trapping is still the more underdefined and strategically valuable frontier

## Tier 3: Important but Not the Immediate Next Main Pass

### 5. Returned Exit / Inward Redirection

Why it matters:

- it is the clearest release-side analogue of structured non-release
- it is likely especially relevant to crystal-like or strongly bounded exits

Why it is not first:

- it is still partly dependent on the earlier reflection-side logic
- the model gains more right now from clarifying glass release and trapping

## Cross-Score Findings

## 1. Glass Release Is the Best Exit Anchor

This is the clearest current conclusion.

`glass -> air` now functions for exit the way:

- `air -> glass`

functions for entry.

It is not perfect, but it is the strongest practical anchor.

## 2. Trapping Is the Best Exit Frontier

This is the main strategic result of the scoreboard.

The programme is now mature enough that the next big gain is likely to come not from another clean success case, but from a better account of:

- why release stalls without simply becoming scattering or return

## 3. Crystal Release Is Important but Slightly Narrower

`crystal -> air` remains a strong case, but it is more specialised.

It looks better suited to a second deepening pass once the broader release logic is more settled.

## 4. Exit-Side Failure Modes Are Now Properly Differentiated

The scoreboard supports keeping these distinct:

- diffuse release
- retained exit
- returned exit

That is a genuine gain. The programme is no longer treating poor release as one undifferentiated outcome.
