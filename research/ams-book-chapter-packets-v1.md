# AMS Book Chapter Packets v1

## Purpose

This file translates the master book synthesis outline into chapter-level build packets.

The outline already established:

- the main narrative spine;
- the canonical reuse stack;
- the primary and supporting sources for each major part.

The next useful step is now practical:

```text
define what each chapter is for,
what it must accomplish,
what canonical sources feed it,
and what should stay out of it
```

That is what this file does.

## Central Claim

The central claim is:

> The book should now be built chapter by chapter from a controlled packet system in which each chapter has a clear argumentative job, a small canonical source set, a defined narrative function, and explicit technical boundaries. This is the most reliable way to convert the current research corpus into prose that is coherent, cumulative, and readable.

More briefly:

```text
Each chapter should solve one part of the ontology, not carry the whole corpus.
```

## Packet Method

Each chapter packet should contain five things:

1. chapter purpose
2. chapter claim
3. primary canonical sources
4. supporting technical sources
5. keep-out rules

This is the current best build method.
