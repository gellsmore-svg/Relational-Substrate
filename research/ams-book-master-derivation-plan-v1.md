# AMS Book Master Derivation Plan v1

## Purpose

This file defines how the assembled manuscript should be turned into the English master book version.

The purpose of the master derivation pass is to produce:

- one stable English master text;
- one clean source for future level-specific versions;
- one governing document that inherits the book's structure rather than the archive's sprawl;
- one reference text from which later bachelor, A-level, GCSE, translation, or appendix-bearing derivatives can be made without re-solving the ontology each time.

The purpose is not to create multiple versions now.

It is:

```text
to establish the stable English master from which all later book versions should derive
```

## Central Claim

The central claim is:

> The assembled manuscript is now strong enough to become the English master version. The remaining task is not conceptual development, but derivational discipline: define the master text, clarify what belongs in the master and what belongs in appendices or derivative editions, and make sure future adaptations inherit the book's core argument, order, and governing vocabulary without re-importing research clutter.

More briefly:

```text
The master book should now be treated as the source, not as another experiment.
```

## What the Master Version Must Be

The English master version must be:

- structurally complete;
- argumentatively stable;
- readable as one book;
- strong enough to stand above derivative editions;
- restrained enough that lower-level versions can simplify without having to reconstruct.

The master version is not a dump of maximum detail.

It is:

```text
the highest stable integrated form of the book's core argument
```

## What the Master Version Must Contain

The master version must contain:

1. the eight-chapter climb already assembled;
2. the governing ontology vocabulary in stable form;
3. the machine -> life -> person ladder;
4. the corruption -> redemption -> resurrection arc;
5. enough material specificity to ground the argument without reopening the archive.

## What the Master Version Must Not Contain

The master version must not contain:

- research-file language;
- scorekeeping language;
- calibration-phase language;
- comparative corpus-control language;
- version-to-version debate residue;
- unnecessary side-branches that are better treated as appendices or derivatives.

Most importantly, the master must not feel like:

```text
a cleaned research archive
```

It must feel like:

```text
a complete book
```

## Master Versus Derivative Rule

The strongest current derivation rule is:

```text
derive downward by controlled simplification, not upward by re-expansion
```

That means:

- the master should stay conceptually full;
- derivative versions should simplify language, pacing, and support burden;
- appendices should carry bounded specialist extensions;
- no later edition should be allowed to redefine the book's main order from scratch.
