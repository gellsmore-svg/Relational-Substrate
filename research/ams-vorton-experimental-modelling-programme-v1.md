# AMS Vorton Experimental Modelling Programme v1

## Purpose

This document reframes the technical AMS work from static ontology mapping into an experimental modelling programme.

The central constraint is simple:

- no one can directly observe a vorton in ordinary empirical practice

That means the work cannot proceed as if the hidden substrate were already visible and only needed description.

Instead, the programme must proceed by:

- proposing model families
- running structured conceptual, geometrical, and mathematical experiments
- comparing the resulting behaviour against observed physical reality
- remodelling where necessary

This is therefore an experimental and iterative programme, not only a classificatory one.

## Governing Commitment

The aim is not to prove the existence of a vorton by direct inspection.

The aim is to discover whether a sufficiently rich but disciplined vorton / torsion / knotting model space can generate coherent explanations for:

- stable material identity
- instability and transient behaviour
- boundary and interface effects
- optical transmission, scattering, absorption, and reflection
- higher-order whole formation

The logic is inferential and experimental:

- if a hidden substrate cannot be seen directly, its candidate structures must be explored by the behaviours they can and cannot generate

## Programme Character

This programme should be understood as:

- brute-force in the positive sense
- exploratory rather than prematurely elegant
- willing to remodel rather than only confirm
- comparative rather than isolated

That means model failure is not wasted effort. It is part of the discovery process.

Some models will:

- fail cleanly
- fail ambiguously
- partially succeed
- reveal hidden parameter needs
- collapse into broader model families
- generate unexpected explanatory advantages

All of those are useful outcomes.

## Core Question Set

The programme is trying to answer questions like:

- what kinds of vorton closure could support stable material identity?
- what kinds of torsion or knotting regime would be too unstable to persist?
- how could a transmissive medium like glass be modelled in vorton terms?
- how could light reflect from metal in vorton terms?
- why do glass, crystal, metal, and polymer differ in transmissive and reflective behaviour?
- how do defects, boundaries, and interfaces alter propagation?
- what kinds of higher-order grouped wholes emerge from repeated local closure?

## Main Principle Of Progress

Progress should come from:

- repeated modelling
- repeated comparison
- repeated remodelling

not from locking one elegant model too early.

This is especially important because many candidate models may be:

- underconstrained
- mathematically equivalent at first pass
- distinguishable only at higher-order or boundary cases

## Programme Structure

The programme should run in six linked layers.

### Layer 1: Model-Space Definition

Define a bounded but expandable space of candidate vorton mechanisms.

This should include:

- closure classes
- torsion classes
- knotting / linking families
- coupling grammars
- slip / reseating modes
- resonance regimes
- boundary interaction modes

The model space must be broad enough to generate variety but narrow enough to remain tractable.

### Layer 2: Behaviour Classes

Define the main behaviours the models must be tested against.

Minimum behaviour classes:

- persistence
- metastability
- collapse
- transmission
- reflection
- absorption
- scattering
- defect sensitivity
- interface transition
- higher-order whole formation

### Layer 3: Material Classes

Run experiments first at the material-class level before full element-by-element saturation.

Priority classes:

- gases
- liquids
- crystals
- glasses
- metals
- polymers
- membranes

### Layer 4: Element And Formation Families

Then deepen by element and formation family.

The point is not to map every element immediately, but to pressure the model space across very different chemistries and whole-forming tendencies.

### Layer 5: Mathematical Formalisation

As the better model families emerge, define:

- variables
- invariants
- stability criteria
- transition conditions
- boundary conditions
- propagation conditions

The maths should emerge from experimental pressure rather than being imposed too early as decoration.

### Layer 6: Comparative Revision

After each modelling round:

- compare models against observed behaviour
- compare models against one another
- compare explanatory fertility, not just local fit
- remodel

This is where the brute-force character of the programme becomes productive.

## Model Families

The first modelling rounds should not try to define one final vorton. They should define families.

Suggested first family set:

### Family A: Simple Closed Torsion Loop

Basic idea:

- local torsion closes on itself
- persistence depends on boundedness and return-path

Use:

- baseline identity model

Main questions:

- too simple?
- enough for local persistence?
- unable to scale into richer boundary behaviour?

### Family B: Distributed Ring Closure

Basic idea:

- closure depends on distributed relation across a ring or loop-like family

Use:

- ring-like grouped persistence
- aromatic or recurrent closure analogies

Main questions:

- does distributed closure give better stability?
- how does it fail?

### Family C: Nested Closure

Basic idea:

- smaller closures participate in a larger enclosing closure

Use:

- higher-order wholeness
- crystal or membrane analogies

Main questions:

- when does nested closure yield true higher-order identity?
- when does it merely produce grouped local stability?

### Family D: Resonant Dynamic Closure

Basic idea:

- persistence depends not on static lock-up alone, but on recurring dynamic coherence

Use:

- water-network-like wholes
- oscillatory or regulated living systems

Main questions:

- when is dynamic closure stable?
- when does it become unstable oscillation instead?

### Family E: Boundary-Concentrated Functional Closure

Basic idea:

- the most important organised behaviour occurs at the interface rather than in uniform bulk

Use:

- catalytic surfaces
- membranes
- metal reflection

Main questions:

- when is the boundary constitutive?
- when is it only secondary?

### Family F: Defect-Responsive Closure

Basic idea:

- the model includes defect, discontinuity, and retained history as part of what the whole is

Use:

- metals
- crystals
- hysteresis
- memory-bearing systems

Main questions:

- how does a whole remain itself while carrying defect history?

## Experimental Classes

These are the main kinds of experiment the programme should run.

### Class 1: Stability Experiments

Ask:

- does this model persist?
- under what conditions?
- what destroys it?

### Class 2: Interaction Experiments

Ask:

- how do two closures interact?
- do they bind, scatter, resonate, destabilise, or ignore each other?

### Class 3: Boundary Experiments

Ask:

- what happens when a propagating or persistent regime meets a boundary?
- what changes when the boundary is rigid, distributed, porous, metallic, or defect-rich?

### Class 4: Optical Experiments

Ask:

- how does transmissive propagation occur through a medium?
- why does a medium transmit, absorb, or scatter?
- why does reflection occur at a metal boundary?

These are especially important because they are among the clearest observable behaviours the programme wants to explain.

### Class 5: Whole-Formation Experiments

Ask:

- when do local closures become higher-order wholes?
- when do they remain aggregates only?

### Class 6: History Experiments

Ask:

- how does prior strain or alignment affect future behaviour?
- can the model generate hysteresis or memory?

## Optical Benchmark Questions

This programme should now explicitly include optical benchmark questions, because these are among the user’s primary target problems.

### Benchmark 1: Transmission Through Glass

Question:

- how does light travel through glass from a vorton perspective?

Relevant pressure points:

- glass as a disordered but real whole
- transmission with limited scattering
- defect sensitivity
- phase delay or relation to whole structure

Model-family candidates likely to matter:

- distributed closure
- dynamic coherence
- boundary and defect interaction

### Benchmark 2: Reflection From Metal

Question:

- how does light reflect from metal from a vorton perspective?

Relevant pressure points:

- metallic whole identity
- boundary-concentrated behaviour
- surface response
- collective rather than isolated local effect

Model-family candidates likely to matter:

- boundary-concentrated functional closure
- defect-responsive closure
- resonant interaction at surface layers

### Benchmark 3: Crystal Versus Glass Transmission

Question:

- why does a crystal transmit differently from glass if both can be transparent?

Relevant pressure points:

- ordered whole versus disordered but real whole
- defect and boundary sensitivity
- propagation through different relation-grammars

### Benchmark 4: Absorption, Scattering, And Defect Effects

Question:

- how do defects, impurities, and interfaces alter propagation?

Relevant pressure points:

- non-persistent topology interaction
- local/global incoherence
- resonant destabilisation

## Geometry And Mathematics

Yes, this programme likely requires mathematics and geometry.

But maths should be introduced in a staged way.

### Stage 1: Geometric Formalisation

Define:

- closure shape
- adjacency structure
- knotting or linking type
- boundary relation
- defect location

### Stage 2: State Variables

Define candidate variables such as:

- torsion magnitude
- closure tightness
- strain distribution
- return-path strength
- coupling strength
- slip tolerance
- resonance compatibility
- boundary impedance

### Stage 3: Transition Rules

Define:

- stability thresholds
- metastability conditions
- collapse conditions
- reflection or transmission conditions
- boundary-triggered changes

### Stage 4: Comparative Mathematics

Use the emerging formalisation to compare:

- which model families can generate the needed behaviours
- which require too many arbitrary assumptions

## Recording Format

Each experiment should record:

- `Model family`
- `Parameter setting`
- `Target behaviour`
- `Observed qualitative outcome`
- `Mathematical or geometrical consequence`
- `Comparison with known physical behaviour`
- `Status`
  - coherent
  - promising
  - degenerate
  - underconstrained
  - interesting anomaly
  - remodel needed

## What Counts As Progress

Progress means:

- identifying model families that generate richer behaviour with fewer arbitrary patches
- discovering which parameters matter most
- learning which behaviours distinguish one family from another
- improving the ability to explain real optical and material cases

## What Counts As Failure

Failure includes:

- models that explain everything only by verbal flexibility
- models that require too many unconstrained parameters
- models that cannot distinguish transmission from reflection in any disciplined way
- models that collapse when defects or boundaries are introduced

## Summary Formula

The point of this programme is not:

- to wait until someone can see a vorton directly

The point is:

- to iteratively brute-force a hidden model space by experimentation, comparison, remodelling, geometry, and mathematics until some families prove more fertile, coherent, and physically explanatory than others

That is the right technical next phase for AMS.
