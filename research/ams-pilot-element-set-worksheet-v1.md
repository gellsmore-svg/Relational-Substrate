# AMS Pilot Element Set Worksheet v1

## Purpose

This worksheet is the first execution layer of the AMS substrate coherence programme.

It turns the research framework into a bounded pilot set that can actually be worked through case by case.

The pilot set is deliberately small enough to govern and broad enough to stress the ontology across very different chemical behaviours.

## Pilot Set

The pilot elements are:

- `H`
- `C`
- `N`
- `O`
- `Na`
- `Mg`
- `Si`
- `P`
- `S`
- `Fe`

## Why These Elements

This set is designed to cover:

- simple bonding
- strong polarity effects
- ionic behaviour
- covalent behaviour
- network formation
- organic centrality
- biological relevance
- metallic and coordination complexity

The pilot is not trying to represent the entire periodic table fairly in one pass. It is trying to reveal whether AMS can survive a diverse first pressure test.

## Pilot Objectives

For each element, the pilot should produce:

1. a representative `T1B` formation set
2. a first-pass AMS vorton relation grammar
3. a first-pass interaction check against non-persistent topology classes
4. one or more candidate `T1C` escalation cases

## General Rules

### Rule 1

Do not begin with AMS language alone.

Each case must begin with conventional chemical description.

### Rule 2

Do not claim explanatory success too early.

Each case must distinguish:

- `Observed`
- `Strong AMS inference`
- `Conjectural AMS extension`
- `Open`

### Rule 3

Do not optimise for quantity in the pilot phase.

The pilot is successful if it produces disciplined comparisons, not if it fills a large table quickly.

## Per-Element Record Template

Each element should be worked through using the same template.

### Element Header

- `Element`
- `Atomic number`
- `Pilot reason for inclusion`
- `Dominant chemical behaviours to test`

### T1B Representative Set

Choose up to `10` representative formations.

For each formation record:

- `Formation name`
- `Formula`
- `Structure type`
- `Bonding mode`
- `Geometry / symmetry`
- `Persistence conditions`
- `Phase tendencies`
- `Reactivity profile`
- `Observed explanatory challenge`
- `AMS closure type`
- `AMS coupling type`
- `AMS lock-up conditions`
- `AMS slip tolerance`
- `AMS strain distribution`
- `AMS return-path viability`
- `Confidence status`

### Non-Persistent Topology Interaction Check

For the same element, ask:

- which non-persistent topology classes are likely to interact with these T1B formations?
- do they destabilise, catalyse, scatter, transiently bind, or trigger threshold change?
- which cases appear most useful for negative-space ontology?

### T1C Escalation Candidates

List the main higher-order formations to test next.

For each one record:

- `Formation type`
- `Why it is a real T1C test`
- `Boundary significance`
- `Persistence significance`
- `Higher-order identity significance`
- `Expected AMS challenge`

## Element Worksheets

## 1. Hydrogen (`H`)

### Pilot reason

- simplest bonding cases
- diatomic baseline
- strong relevance to polarity, proton behaviour, water, acids, and hydrogen bonding

### Dominant behaviours to test

- diatomic persistence
- polar bonding
- hydrogen bonding
- proton transfer
- extreme simplicity versus wide relational reach

### Candidate T1B set

- `H2`
- `H2O`
- `HCl`
- `NH3`
- `CH4`
- `HF`
- `H3O+`
- `OH-`

### Candidate T1C escalation cases

- water networks
- acid-base systems
- hydrogen-bond-dominated assemblies
- membrane or protein-relevant hydrogen-bond regimes

### Initial AMS pressure points

- how minimal closure participates in strong relational effects
- how local simplicity can support large-scale structured behaviour
- how transient proton-related states fit the stability / instability map

## Pilot Sequence

The pilot should run in this order:

1. `H`
2. `C`
3. `O`
4. `N`
5. `Si`
6. `Fe`
7. `Na`
8. `Mg`
9. `P`
10. `S`

This sequence starts with the strongest baseline explanatory cases, then moves toward the cases most likely to stress higher-order and threshold reasoning.

## Pilot Exit Criteria

The pilot phase should only be considered complete when:

- each pilot element has a representative T1B set
- each set has at least first-pass AMS relation grammar
- each element has been checked against non-persistent topology classes
- each element has at least one serious T1C escalation target
- a real problems list exists, not just positive claims
