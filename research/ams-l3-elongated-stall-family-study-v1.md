# AMS L3 Elongated Stall Family Study v1

## Purpose

This document isolates the strongest recurring stall family identified so far:

- `L3` elongated stall loop

From:

- `ams-trapped-release-loop-comparison-v1.md`

the main result was that `L3` is the first loop geometry that appears to recur across multiple high-value contexts rather than belonging to only one narrow case.

It appears in:

- glass release
- crystal release
- glass-to-crystal compatibility stall

The purpose of this study is therefore to ask:

- what makes an elongated stall possible?
- why does it recur?
- what determines whether it resolves into handoff, trapping, return, or fragmentation?

## Core Claim

`L3` is not just “a bigger loop.”

It is a distinct interface-stall family in which:

- the event progresses partway through a terminal or transition layer
- mediated continuity remains real for a time
- full completion fails before outward release or cross-whole admission resolves
- local reseating continues in an elongated path rather than collapsing immediately into shallow face retention or diffuse breakdown

That makes `L3` the first candidate for a general interface-stall geometry in the AMS programme.

## Working Definition of `L3`

An elongated stall loop is a bounded but non-shallow interface event in which:

- the event penetrates beyond the immediate boundary face
- progression through the terminal or transition layer begins
- local reseating remains organised enough to preserve a recognisable path
- final completion fails
- the event therefore cycles or stalls along an extended local route

In short:

- partial progression without completion

## Minimal Conditions for `L3`

The current model suggests five minimum conditions.

## 1. A Finite Active Layer

`L3` requires a real terminal or transition layer.

If the interface were only a zero-thickness face, there would be no spatial depth in which elongated stall could occur.

So `L3` itself supports the claim that:

- some interfaces are finite active regions

not merely surfaces.

## 2. Non-Trivial Through-Layer Continuity

`L3` requires more than shallow boundary response.

That means:

- `J_bound` alone is not enough
- `J_trans` must be real

But that continuity must be:

- insufficient for completion
- sufficient for extended local progression

So `L3` lives in an intermediate regime:

- `J_trans` neither too low nor high enough for success

## 3. Partial Coherence Preservation

If coherence were too weak, the event would fragment toward `L4`.

If coherence were too strong and well aligned, the event would likely complete or return more cleanly.

So `L3` appears to require:

- medium or medium/high `Q_coh`

not the highest possible coherence, but enough to preserve a recognisable path.

## 4. Incomplete Escape

`L3` requires:

- some viable outward or onward possibilities

otherwise the event would remain in shallow retention only.

But those possibilities must not dominate.

So `L_escape` is likely:

- medium

That is one of the clearest internal signatures of `L3`.

## 5. Controlled Obstruction

`L3` appears to require obstruction strong enough to prevent completion, but not so strong that the event immediately degrades into broken multi-loop behaviour.

That means:

- `D` is often medium
- or medium/high but not maximally destructive

This again places `L3` in an intermediate regime rather than at an extreme.

## Best Current Variable Profile

Across the current cases, the best provisional profile for `L3` is:

- `B_in`: medium when admission is relevant
- `B_out`: low/medium when release is relevant
- `J_bound`: medium/high
- `J_trans`: medium
- `Q_coh`: medium or medium/high
- `D`: medium
- `I_comp`: medium where cross-whole compatibility is involved
- `L_depth`: medium
- `L_span`: medium/high
- `L_escape`: medium

This profile is not yet universal, but it is the clearest common cluster so far.

## Why `L3` Recurs Across Contexts

The recurrence is not accidental.

The common structure is:

- a real event begins transition
- the interface is active enough to mediate that transition
- the path is long enough to develop internal organisation
- final completion fails

That logic can appear in:

### Glass Release

- the event begins terminal release
- the terminal layer supports some progression
- clean outward emergence fails

### Crystal Release

- the event approaches release in a more patterned internal form
- conversion begins
- ordered release does not complete

### Glass->Crystal Compatibility

- distributed propagation enters the compatibility zone
- narrowing toward patterned admission begins
- full crystal-style handoff fails

So the same family recurs because the same structural condition recurs:

- partial, mediated transition that never finishes

## `L3` Compared with Other Loop Families

## Compared with `L1`

`L1` is too shallow.

It lacks:

- real depth
- meaningful transition-layer progression

So `L1` fits immediate surface retention, not extended stall.

## Compared with `L2`

`L2` supports bounded local retention in a shallow pocket.

It can resemble `L3`, but the difference is:

- `L2` is pocketed and locally contained
- `L3` is path-like and progression-bearing

So `L3` is better for cases where transition genuinely begins.

## Compared with `L4`

`L4` is what happens when `L3` loses enough organisation to fragment.

That means `L4` is not a separate peer in the same sense. It is often a degradation path out of `L3`.

This relation is important.

## Four Main Resolution Paths of `L3`

The main value of isolating `L3` is that it now lets the programme ask:

- what does an elongated stall become?

Current answer: four main possibilities.

## Path R1: Successful Handoff

### Description

The elongated stall does not remain stalled. Mediated continuity eventually becomes sufficient for completion.

### What Must Improve

- `J_trans` rises
- `L_escape` rises
- `Q_coh` stabilises or strengthens
- local obstruction is reduced or reorganised

### Result

- release or compatibility transition completes

## Path R2: Trapped Release

### Description

The elongated path remains active but completion never dominates. The event settles into persistent local release failure.

### What Maintains It

- `J_trans` remains medium rather than high
- `B_out` stays low/medium
- `Q_coh` remains sufficient for bounded local organisation
- `D` prevents completion without destroying the path

### Result

- prolonged terminal-layer stall

## Path R3: Ordered Return

### Description

The elongated path resolves not into outward completion but into stronger directional return.

### What Must Shift

- `J_bound` strengthens relative to `J_trans`
- `Q_coh` remains high enough for directional organisation
- the boundary increasingly favours inwardly coherent redirection

### Result

- the stalled event resolves into structured non-release rather than local stall

## Path R4: Broken Multi-Loop Fragmentation

### Description

The elongated path loses enough continuity or coherence that it breaks into multiple weaker reseating regions.

### What Must Degrade

- `Q_coh` falls
- `J_trans` weakens
- `D` rises or becomes more destructively distributed
- `L_escape` remains poor

### Result

- `L4` style breakdown toward scattering or incoherent failure

## What Controls the Resolution Path

The current model suggests four main control factors.

## 1. Balance Between `J_bound` and `J_trans`

This is one of the strongest current discriminators.

- higher `J_trans` favours completion
- stronger `J_bound` relative to `J_trans` favours local retention or return

## 2. Stability of `Q_coh`

If coherence remains strong enough, the elongated path can remain organised.

If it weakens too much, the path breaks toward `L4`.

## 3. Character of Obstruction

Not only how much obstruction exists, but how it is distributed seems to matter.

An event can survive obstruction if it is:

- localised
- structured
- or gradual

It is more likely to fragment if obstruction is:

- widely disruptive
- irregular
- or rapidly varying across the layer

## 4. Availability of Escape Routes

If no escape routes exist, the event may collapse into shallower retention.

If too many weak escape routes exist, the event may diffuse.

If a few viable but difficult escape routes exist, `L3` becomes especially plausible.

That makes `L_escape` one of the most interesting geometric descriptors in the whole current programme.

The key gains are still coming from:

- better geometry

not from multiplying variables.

## Current Weak Points

## 1. `L_escape` Is Useful but Still Informal

It may eventually need more explicit treatment.

For now it works best as a geometric descriptor rather than a formal variable.

## 2. Ordered Return Still Needs Direct Comparison Against `L3`

The current study says how `L3` can resolve into return, but a dedicated side-by-side ordered-return study would still be useful.

## 3. Obstruction Character Needs Better Modelling

The study now clearly suggests that obstruction distribution matters, not only obstruction magnitude.

That remains under-modelled.
