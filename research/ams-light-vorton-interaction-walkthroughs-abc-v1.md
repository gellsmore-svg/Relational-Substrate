# AMS Light-Vorton Interaction Walkthroughs A/B/C v1

## Purpose

This note gives step-by-step walkthroughs for a light-like transient encountering a stable vorton candidate.

It uses three options:

```text
Option A: admitted/transmitted
Option B: returned/reflected
Option C: stored/scattered/converted
```

The aim is conceptual clarity. Light is not being made into a vorton. A vorton is not being made into matter-stuff. This is an admissibility walkthrough.

## Guardrail

The corrected AMS chain remains:

```text
continuous substrate
-> local organised disturbance
-> torsion/tension/alignment expression
-> candidate closure-pattern
-> stable primary topology/vorton
-> coupled vorton regimes
-> secondary material regimes
-> perceived matter
```

Light-like propagation belongs to transient structure unless it crosses closure thresholds. It is normally:

```text
open, coherent, route-bearing, unwindable
```

A stable vorton is:

```text
closed or quasi-closed, bounded, returnable, invariant-preserving
```

## Shared Starting Conditions

A light-like transient approaches a stable vorton candidate or vorton-bearing regime.

Represent the transient by:

```text
L = [coherence, frequency, phase relation, amplitude, route direction, polarisation-like orientation]
```

Represent the vorton candidate by:

```text
V = [C_class, W_sig, H_sig, R_map, B_env, Q_band, L_band, S_env, K_couple, M_reseat, F_modes]
```

The interaction asks:

```text
is L admissible through V or the V-bearing regime?
```

Admissibility is not all-or-nothing. The transient may split.

## Option A: Admitted / Transmitted

### Step A1: Approach

The light-like transient reaches the boundary of a vorton-bearing regime.

```text
L approaches V-boundary
```

### Step A2: Boundary Compatibility Test

The boundary relation checks whether the transient can be handed through without breaking coherence.

AMS variables:

```text
J_bound high
J_trans high
Q_coh preserved
K_store low
X_frag low
```

### Step A3: Vorton Reseating Is Not Required Or Is Tolerable

The closure-pattern either does not need to reseat significantly, or its permitted reseating modes absorb the route pressure without identity stress.

```text
M_reseat compatible
B_env not threatened
R_map not overloaded
```

### Step A4: Delay Without Capture

The transient is admitted but delayed.

This corresponds to:

```text
coherent propagation through a route-bearing regime
```

Classical analogue:

```text
transparent material with refractive delay
```

AMS reading:

```text
the transient remains open and coherent,
the vorton-bearing regime supports route continuity,
and stable closure is not converted into transient motion.
```

### Step A5: Exit

The transient exits with changed timing, phase, or direction, but still as light-like propagation.

```text
L_in -> admitted route -> delayed L_out
```

### Verdict

```text
light remains non-vorton
vorton remains stable
interaction = transmission/admission
```

## Option B: Returned / Reflected

### Step B1: Approach

The transient reaches a boundary whose closure relation is not deeply admissive.

```text
L approaches V-boundary
```

### Step B2: Boundary Mismatch

The incoming route structure cannot be handed into the regime without violating boundary compatibility.

AMS variables:

```text
J_bound low or asymmetric
J_trans into regime low
boundary return high
K_store low to moderate
X_frag low if reflection is coherent
```

### Step B3: Closure Pattern Resists Admission

The vorton-bearing regime does not need to be ordinary matter-stuff to constrain passage.

It constrains because:

```text
its closure profile has admissibility conditions
```

The transient cannot enter without creating inadmissible route stress.

### Step B4: Return Path Forms

The transient is redirected or returned.

Classical analogue:

```text
reflection at a boundary
```

AMS reading:

```text
route incompatibility produces boundary return
```

### Step B5: Coherent Or Diffuse Return

If boundary order is high:

```text
coherent reflection
```

If boundary order is poor:

```text
diffuse scattering
```

### Verdict

```text
light remains non-vorton
vorton remains stable
interaction = return/reflection
```

## Option C: Stored / Scattered / Converted

Option C covers cases where the interaction does not cleanly transmit or cleanly return.

### Step C1: Approach

The transient reaches a regime with partial compatibility.

```text
L approaches V-bearing boundary
```

### Step C2: Partial Admission

Some of the transient is admitted, but it cannot continue as a clean coherent route.

AMS variables:

```text
J_bound partial
J_trans uneven
K_store rising
Phi_lag rising
X_frag rising
L_leak rising
```

### Step C3: Storage

Part of the incoming route pressure is stored as local stress, polarisation-like lock-up, or delayed release.

```text
L portion -> stored mismatch
```

This is not vorton generation unless closure thresholds are crossed.

### Step C4: Scattering

If local compatibility varies across the regime, the transient fragments into multiple route outcomes.

```text
L portion -> scattered routes
```

Classical analogue:

```text
rough surface, cloudy medium, defect-rich material
```

AMS reading:

```text
route compatibility is locally inconsistent
```

### Step C5: Conversion

If the transient couples into material modes, it may become heat-like loss, local vibration, electrical response, or other secondary regime activity.

```text
L portion -> material route mode
```

This is absorption or conversion, not stable vorton identity.

### Step C6: Proto-Closure Possibility

Only in a special threshold case can the interaction seed a proto-vorton.

Required:

```text
B_in rises above threshold
D_spread sufficient
R_return emerges
Q_coh remains high
L_leak below threshold
I_profile begins to persist
```

Without these, the event is storage, scattering, or conversion only.

### Verdict

```text
ordinary case: light remains non-vorton, interaction = storage/scattering/conversion
special threshold case: possible proto-vorton, not stable vorton unless I_profile persists
```

## Why This Helps

The walkthrough clarifies the main distinction:

```text
non-vorton propagation can be real, coherent, and powerful
without being stable primary topology.
```

It also clarifies why a vorton can constrain transmission without being ordinary matter:

```text
constraint comes from admissibility conditions of stable topology.
```

```text
stored, scattered, converted, or, rarely, drawn into a new closure.
```
