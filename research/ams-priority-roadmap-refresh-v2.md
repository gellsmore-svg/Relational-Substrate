# AMS Priority Roadmap Refresh v2

## Purpose

This note refreshes the earlier research roadmap in light of the latest work completed since `v1`.

Since the earlier roadmap, the programme has materially advanced in two major directions:

- formal substrate scaffolding;
- arc-level comparative hardening.

That changes the priority order.

The question is no longer simply:

```text
what was weakest before formalisation began?
```

It is now:

```text
what remains weakest after the latest gains,
and what most strengthens the programme next?
```

## What Has Improved Since Roadmap v1

The following areas are materially stronger now.

## 1. Substrate Formal Scaffold

New files:

- `ams-vorton-geometry-and-closure-formalism-v1.md`
- `ams-route-variable-mathematical-framework-v1.md`
- `ams-topology-transition-criteria-v1.md`
- `ams-formalism-to-element-family-mapping-v1.md`

Improvement:

The programme no longer lacks a formal scaffold entirely.

It now has:

- primitive objects;
- closure criteria;
- route-state vector;
- transition classes;
- first formal bridge back to family identities.

This does not finish the substrate problem, but it significantly improves it.

## 2. Family Hardening

New file:

- `ams-iron-non-heme-catalytic-route-map-v1.md`

Improvement:

One of the clearest empirical family gaps is now closed, and iron has been recalibrated upward in:

- `ams-iron-family-score-recalibration-v2.md`

This strengthens the empirical side materially.

## 3. Boundary Hardening

New files:

- `ams-life-boundary-edge-cases-v1.md`
- `ams-personhood-edge-cases-v1.md`

Improvement:

The machine/life/person ladder is now less vulnerable at the margins.

This is a major gain because one of the main higher-level weaknesses was boundary brittleness.

## 4. Arc-Level Comparative Hardening

New files:

- `ams-corruption-case-comparison-matrix-v1.md`
- `ams-restoration-versus-replacement-comparison-v1.md`
- `ams-resurrection-continuity-stress-test-v1.md`

Improvement:

The arc-level claims are now more comparative and less purely synthetic.

This is one of the biggest conceptual upgrades since the previous roadmap.

## What Remains Weakest Now

After these gains, the weakest remaining areas have shifted.

## 1. Formal Calibration, Not Formal Absence

The problem is no longer:

```text
there is no formal scaffold
```

The problem is now:

```text
the scaffold is not yet calibrated, quantified, or tied tightly enough to measured or strongly constrained system cases
```

This is a real improvement in problem quality.

## 2. Edge-Case Integration into the Main Thesis

The life and personhood boundaries are stronger, but those edge-case results are not yet fully integrated back into:

- the main full-thesis synthesis;
- the arc-level summaries;
- the machine/life/person ladder at summary level.

The danger is not incoherence, but drift between summary files and hardened edge-case files.

## 3. Remaining Family Edge Cases

Iron improved.

Still weaker than the strongest families:

- sulfur in its most context-sensitive extensions;
- oxygen at large planetary / atmospheric integration scale;
- phosphorus in pathological mislocation and more complex transfer logic.

## 4. Quantitative Machine Comparison

The machine work is strong qualitatively and comparatively.

What remains weaker is:

- dependency matrices;
- state-transition modelling;
- failure propagation formalisation;
- sharper quantitative discrimination between machine classes.

## 5. Social / Ecclesial / Civilisational Extension

This remains the highest-risk outward extension zone.

It should still not be prioritised too early.

But because the arc-level work is now stronger, it is becoming more approachable, provided the next moves remain disciplined.

## Updated Priority Principle

The principle should now be refined from:

```text
increase constraint before scope
```

to:

```text
increase calibration before extension
```

The reason is simple:

The programme now has a real scaffold.
What it lacks most is sharper calibration and integration.

## Refreshed Priority Order

The new recommended research order is:

```text
1. Calibrate the formal scaffold against stronger system cases
2. Close the remaining high-value family edge cases
3. Quantify machine dependency and failure propagation
4. Integrate edge-case results back into the summary ladder
5. Only then extend outward into broader social/ecclesial domains
```

## Priority 1: Formal Calibration Against Strong Cases

### Why first

The substrate work has crossed from absence to scaffold.

The next highest-value gain is to make it answerable to the strongest empirical-comparative cases.

### Best target cases

- respiration / ATP synthesis;
- ribosome / translation;
- oxygen route families;
- iron route families.

### Deliverable targets

Best next files in this lane:

```text
ams-core-runtime-machines-quantitative-comparison-v1.md
ams-respiration-failure-propagation-matrix-v1.md
ams-ribosome-route-dependency-matrix-v1.md
```

### Why it matters

This is the fastest route from semi-formal language toward constrained model discrimination.

## Priority 2: Remaining Family Edge Cases

### Why second

The family layer remains one of the best assets in the programme.

That is exactly why remaining edge cases should be tightened before adding many new families.

### Best remaining targets

```text
ams-sulfur-sulfation-pattern-specificity-stress-test-v1.md
ams-oxygen-planetary-scale-source-sink-stress-test-v1.md
ams-phosphorus-pathological-mislocation-and-calcification-v1.md
```

### Why it matters

If the strongest families survive their hardest edge cases, the comparative core becomes much harder to dismiss.

## Priority 3: Quantified Machine Failure and Dependence

### Why third

The machine files are now among the strongest comparative parts of the programme.

The next gain is to make them more explicit in:

- dependency order;
- failure propagation;
- variable-state shifts;
- threshold behaviour.

### Deliverable targets

```text
ams-respiration-failure-propagation-matrix-v1.md
ams-ribosome-route-dependency-matrix-v1.md
ams-core-runtime-machines-quantitative-comparison-v1.md
```

### Why it matters

These files would convert the strongest machine claims from persuasive structure into more constrained system analysis.

## Priority 4: Summary-Level Reintegration

### Why fourth

The project now has:

- summary syntheses;
- edge-case hardening files;
- formal scaffold files;

but these layers have not yet all been reintegrated into the highest-level summaries.

### Best target files to refresh later

```text
ams-full-ontology-core-thesis-synthesis-v2.md
ams-priority-research-roadmap-v3.md
ams-c-n-p-s-fe-o-cross-family-synthesis-v2.md
```

### Why it matters

Without reintegration, the corpus becomes stronger locally but uneven globally.

## Priority 5: Controlled Outward Extension

### Why fifth

This remains high-risk and should still be delayed.

But after the new hardening work, it is closer than it was before.

### Conditions for safe extension

Do not prioritise outward extension until:

- machine comparison is more calibrated;
- remaining family edge cases are tightened;
- top-level syntheses are refreshed with the new boundary and arc work.

### Possible later targets

```text

ams-ecclesial-order-and-false-persistence-v1.md
```
