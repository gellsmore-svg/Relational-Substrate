# AMS Obstruction Character Study v1

## Purpose

This document develops the strongest current next frontier in the AMS interface programme:

- obstruction character

The need for this study emerged from:

- `ams-route-reality-integration-note-v1.md`

The current programme already models:

- interface variables
- loop families
- route classes
- route-reality grades

well enough that the main remaining weakness is no longer “what variables do we need?” but rather:

- what kind of obstruction is acting on the event?

The core claim here is that obstruction should not be treated only by amount. Its distribution, shape, and relation to the active route matter just as much.

## Core Claim

Obstruction is not one thing.

Different obstruction characters do different kinds of work:

- some preserve one narrow viable route while blocking others
- some create shallow retention and pseudo-routes
- some bias the event toward return
- some fragment the route field entirely

So obstruction must be treated not only as:

- how much blockage exists

but as:

- how that blockage is arranged relative to route progression

## Current Programme Relevance

Obstruction character now sits at the intersection of:

- `L3` elongated stall
- `R3` blocked apparent route
- `R5` loop-dominated pseudo-route
- `L_escape`
- boundary-layer degradation

That means this is not a side topic. It is now central to understanding why the same interface variables can resolve into very different geometric outcomes.

## Obstruction Character vs Obstruction Magnitude

Two cases may have similar total obstruction burden and still behave very differently.

For example:

- one may preserve a narrow viable route
- another may fragment every route into weak local fields

So obstruction magnitude alone cannot explain route outcome.

This suggests a working distinction:

- magnitude asks how much obstruction exists
- character asks how obstruction is spatially and functionally arranged

## First-Pass Obstruction Character Classes

## Class O1: Funnelling Obstruction

### Description

Obstruction blocks many possible continuations while preserving or even accentuating one narrow viable route.

### Main Effect

- reduces route count
- increases route selectivity
- can improve completion if one route remains genuinely viable

### Likely Associations

- `R1` narrow viable route
- successful `glass -> crystal` handoff in selective zones
- some cleaner crystal release cases

### Main Risk

If the remaining route is not truly viable, funnelling obstruction can become:

- `R3` blocked apparent route

## Class O2: Late-Gate Obstruction

### Description

A route remains plausible through much of its progression, but obstruction appears or becomes decisive late in the path.

### Main Effect

- preserves apparent route reality for longer
- causes failure near completion rather than near first contact

### Likely Associations

- `R3`
- late-stage release failure
- compatibility paths that almost complete and then fail

### Main Risk

This is one of the strongest generators of misleading near-completion.

## Class O3: Pocketing Obstruction

### Description

Obstruction shapes the active region into bounded local pockets that preserve activity but weaken onward continuity.

### Main Effect

- favours shallow retention
- supports `L2` pocket loops
- encourages pseudo-routes or trapped release

### Likely Associations

- `R5`
- glass terminal retention
- local pseudo-release activity

### Main Risk

The event can look rich in activity while having poor true route reality.

## Class O4: Biasing Obstruction

### Description

Obstruction is arranged in a way that weakens outward or onward completion more than inward or return-side redirection.

### Main Effect

- favours directional asymmetry
- preserves organised return more than completion

### Likely Associations

- `R4`
- polished metal reflection
- ordered return in crystal-like exits

### Main Risk

If described badly, this can be mistaken for generic “strong boundary response” instead of asymmetrically arranged obstruction.

## Class O5: Diffusing Obstruction

### Description

Obstruction does not clearly preserve one route or one bounded pocket. Instead it spreads the event laterally or weakens continuation enough to broaden the release.

### Main Effect

- weakens clean completion
- allows degraded outward continuation
- favours diffuse release over trapping or return

### Likely Associations

- `R2`
- diffuse release in glass exit
- weak distributed continuation without dominant route

### Main Risk

This can easily shade into stronger fragmentation if route coherence falls further.

## Class O6: Fragmenting Obstruction

### Description

Obstruction is distributed in a way that prevents any one route or loop from remaining dominant.

### Main Effect

- breaks route coherence
- destabilises elongated stall
- favours `R6` and exit-side scattering

### Likely Associations

- oxidised surfaces
- broken multi-loop fields
- degraded interface zones

### Main Risk

If overused, this becomes a catch-all. It should be used only where route logic genuinely breaks apart.

## Obstruction Character Table

| Obstruction class | Route effect | Loop effect | Main likely route class | Main likely outcome |
|---|---|---|---|---|
| O1 funnelling | narrows to one route | suppresses weaker loops | `R1` or `R3` | selective handoff or blocked near-completion |
| O2 late-gate | preserves route until late failure | loops secondary | `R3` | blocked apparent route |
| O3 pocketing | weakens onward route, preserves local activity | strengthens `L2` / local retention | `R5` | trapped release / pseudo-route |
| O4 biasing | redirects directional viability | can preserve ordered redirection | `R4` | inward return / reflection |
| O5 diffusing | weakens route dominance without total collapse | low loop centrality | `R2` | diffuse release |
| O6 fragmenting | destroys route dominance | breaks loops into field behaviour | `R6` | scattering / fragmented failure |

## Strongest Distinctions

## O1 vs O2

Both can produce route-like behaviour, but:

- `O1` preserves one narrow route
- `O2` preserves a route for a while and then blocks it

This is the difference between:

- selective viability
- late failure

## O2 vs O3

This is one of the most important distinctions in the whole document.

- `O2` supports `R3`
- `O3` supports `R5`

That means:

- late-gate obstruction creates failed real routes
- pocketing obstruction creates pseudo-routes dominated by local cycling

This may become the cleanest geometric interpretation of the `R3` / `R5` split.

## O4 vs O1

Both can preserve strong structure, but:

- `O1` preserves completion-side selectivity
- `O4` preserves inward or return-side selectivity

That is a major directional distinction.

## O5 vs O6

This is the difference between:

- degraded but still outwardly resolving release
- and route-field collapse

So:

- `O5` fits diffuse release
- `O6` fits fragmentation and scattering

## Best Current Benchmark Mapping

## Air -> Glass

Main obstruction characters:

- weak `O1`
- some `O5` in degraded entry

Interpretation:

- glass entry usually needs permissive continuation more than heavy obstruction analysis, but funnelling can still matter in selective local cases

## Glass -> Air

Main obstruction characters:

- `O2`
- `O3`
- `O5`

Interpretation:

- late-gate obstruction creates blocked release routes
- pocketing obstruction creates pseudo-release loops
- diffusing obstruction creates broad degraded release

This is the richest current obstruction benchmark.

## Air -> Polished Metal

Main obstruction character:

- `O4`

Interpretation:

- the surface is not simply blocking; it is asymmetrically biasing the route field toward return

## Air -> Oxidised Metal

Main obstruction characters:

- `O2`
- `O6`

Interpretation:

- some routes persist long enough to appear promising
- deeper coherence breaks and route fields fragment

## Glass -> Crystal

Main obstruction characters:

- `O1`
- `O2`
- `O3`

Interpretation:

- selective compatibility zones can preserve one viable route
- many routes fail late under real progression
- some compatibility activity becomes pseudo-progressive local looping

This is the strongest cross-whole obstruction benchmark.

## Crystal -> Air

Main obstruction characters:

- `O1`
- `O2`
- `O4`

Interpretation:

- ordered release often preserves selective route logic
- failure tends more toward late blocking or return bias than toward pseudo-route looping

## Programme-Level Gains

## 1. Obstruction Is No Longer Just Quantity

This is the main result.

The programme now has a first-pass obstruction-character taxonomy rather than only a generic sense of blockage or burden.

## 2. The `R3` / `R5` Distinction Now Has a Geometric Interpretation

This is especially important.

Current best reading:

- `R3` is often driven by `O2` late-gate obstruction
- `R5` is often driven by `O3` pocketing obstruction

That is a real explanatory step forward.

## 3. Directional Asymmetry Is Cleaner

`O4` biasing obstruction helps explain why some cases preserve:

- inward return

rather than:

- completion

without treating that as a vague special case.

## 4. Diffuse Release and Fragmentation Are Better Separated

The distinction between:

- `O5`
- `O6`

supports the existing separation between:

- diffuse release
- scattering / fragmented failure

## Current Weak Point

The current taxonomy is strong enough to use, but still needs one harder test:

- a direct obstruction-character benchmark table

That is the obvious next move if the taxonomy is going to become operational rather than just descriptive.

## Confidence

- confidence that obstruction character is now a necessary layer in the programme: `9/10`
- confidence that O1-O6 is a useful first-pass taxonomy: `8/10`
- confidence that O2 and O3 are especially important frontier classes: `9/10`
- confidence that O4 materially clarifies return-side cases: `8/10`

Keep the current primitive variable set unchanged.

Use obstruction character as a modelling layer on top of the existing interface and route framework.
