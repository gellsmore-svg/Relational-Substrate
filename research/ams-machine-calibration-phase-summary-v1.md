# AMS Machine Calibration Phase Summary v1

## Purpose

This file summarises the machine-calibration phase that followed the corpus-state review.

The aim is to answer four questions clearly:

```text
1. What did this phase add?
2. What did it materially improve?
3. What remains unresolved?
4. What should happen next?
```

This is a phase-control summary, not a new independent synthesis.

## Central Claim

The central claim is:

> The machine-calibration phase materially strengthened the AMS corpus by turning its best machine examples into threshold-disciplined cases, clarifying how throughput, coherence, load, competition, diffusion, and fragmentation interact at machine scale, and tightening the bridge from local machine stress to whole-cell state change.

More briefly:

```text
AMS now understands its core machines less as examples and more as threshold-bearing systems.
```

## What Materially Improved

This phase improved the corpus in five main ways.

## 1. Variable Discipline Improved

The route-state variables were already defined, but they were not equally calibrated.

This phase sharpened the practical role of:

- `Q_coh`
- `J_trans`
- `B_out`
- `I_comp`
- `L_diff`
- `X_frag`

The main gain here is that these variables are now less likely to remain abstract labels and more likely to function as discriminants between machine states.

## 2. Respiration Became a Stronger Collapse Model

Respiration was already a strong comparative machine.

This phase made it stronger by defining:

- coherent productive respiration;
- compensated stress respiration;
- thresholded unstable respiration;
- fragmenting respiratory collapse.

The key gain is that respiration now has a more explicit threshold grammar for:

- high throughput versus dangerous overdrive;
- leak versus fragmentation;
- local burden versus machine-order loss.

## 3. Translation Became a Stronger Drift Model

Translation was already well described, but it had not been forced as tightly through threshold logic.

This phase made it stronger by defining:

- productive faithful translation;
- compensated drift translation;
- thresholded misreading and burden propagation;
- renewal collapse translation.

The key gain is that the corpus now distinguishes more sharply between:

- activity

and

- architecture-renewing activity.

That is a major improvement in the machine ontology.

## 4. Machine Diversity Was Better Preserved

One of the main risks in formal work is flattening different machines into one collapse model.

This phase avoided that.

Respiration now reads primarily as:

- pressure-loaded;
- leak-sensitive;
- oxidatively fragmentable.

Translation now reads primarily as:

- fidelity-loaded;
- ambiguity-sensitive;
- burden-accumulating;
- slower to collapse but easier to drift.

That is one of the strongest gains of the phase because it proves the variable scaffold can stay machine-sensitive.

## 5. The Life Bridge Tightened

This phase materially improved the bridge from:

```text
machine
to
cell state
to
life boundary
```

The main gain is that whole-cell failure is now described less as:

```text
one machine breaking
```

and more as:

```text
multiple exported burdens becoming globally owning
```

That is both biologically and ontologically stronger.

## What Remains Unresolved

Even after this phase, several weaknesses remain.

## 1. The Calibration Is Still Comparative, Not Measured

The phase improved calibration discipline, but the assignments still remain:

- comparative;
- semi-formal;
- band-based;
- non-experimental.

That is acceptable at this stage, but it remains a limit.

The next gains would come from tighter weighting logic, not from pretending the current values are experimentally established.

## 2. Cross-Machine Coupling Still Needs More Depth

The bridge file improved the transition from machine to cell state, but:

- the coupling remains qualitative;
- the export channels are not yet weighted;
- the timing of cascade propagation remains broad.

This is the clearest next technical deepening path if calibration is to continue.

## 3. The Formal Scaffold Still Needs Reabsorption Upward

The machine phase improved the middle of the ontology.

But the higher summary layer still needs to absorb these gains more fully.

In particular, the strongest follow-on reintegration targets are:

- the full ontology thesis file;
- the corpus-state summary layer;
- the living-state boundary summary language.

## 4. Iron Still Remains the Main Family-Reconciliation Question

The machine phase did not change the family standings directly.

That means the main remaining family-level decision still concerns whether iron should remain:

```text
9.25/10 provisional upward revision
```

or whether enough cumulative calibration now exists to justify a stronger lock.

This is still unresolved.

## What Should Happen Next

There are now two good next directions.

## Direction A: Deepen Calibration Further

This would mean:

- weighting export channels more explicitly;
- refining cross-machine coupling;
- tightening machine-to-cell propagation logic;
- possibly adding one more comparative calibration file.

This is the more technical continuation.

## Direction B: Reintegrate Upward

This would mean:

- refreshing top-level synthesis files;
- pulling the machine-calibration gains into the main ontology summaries;
- making sure the higher-order claims now reflect the stronger middle layer.

This is the more architectural continuation.

Both are legitimate.

If forced to choose, the better immediate move is:


because the machine layer has just improved materially and the summary layer is now behind it.
