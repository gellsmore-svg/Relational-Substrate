# AMS Machine Calibration and Thresholds Phase v1

## Purpose

This file defines the next focused phase of work for the AMS corpus.

The purpose is not to generate more broad ontology. The purpose is to discipline the strongest existing ontology by forcing it through tighter machine-level calibration.

The phase is controlled by three questions:

```text
1. Which variables now need tighter calibration?
2. Which thresholds matter most?
3. Which machine cases will generate the strongest new constraints?
```

This is therefore a phase-control note, not a new free-standing synthesis.

## Central Claim

The central claim is:

> The most valuable next gains in the AMS corpus will come from calibrating the machine layer more tightly against threshold behaviour, because the machine layer is where the formal scaffold, the element-family identities, and the living-state ontology all meet under enough constraint to expose weak assumptions.

More briefly:

```text
The next phase should harden AMS by calibrating machine thresholds.
```

## Why This Phase Comes Next

The corpus has already achieved three major things:

- a strong six-family comparative core;
- a first semi-formal substrate and route scaffold;
- two well-mapped runtime machines with failure propagation analysis.

That changes the programme's needs.

The main weakness is no longer:

```text
we do not yet have categories
```

The main weakness is now:

```text
we do not yet know tightly enough where machine coherence breaks, reconfigures, compensates, or fragments
```

That is a threshold problem.

Threshold discipline matters because the AMS corpus is not trying merely to classify parts. It is trying to describe when real bounded continuity remains viable, when it drifts, when it compensates, and when it breaks.

That means the next phase must sit at the point where:

- formal variables can be compared;
- family roles can be loaded into real systems;
- failure is not theoretical but propagation-sensitive;
- higher claims about life and corruption remain answerable to real machine behaviour.

The machine layer is the tightest place where all of those demands meet.

## Phase Objective

The objective of this phase is:

```text
to convert the current machine work from good comparative description into a more threshold-disciplined calibration layer
```

That requires four outcomes:

1. stronger variable assignment discipline;
2. clearer threshold relations between variables;
3. better distinction between metastable drift and fragmentation;
4. stronger mapping from local machine failure to cell-state consequences.

## The Main Variables That Need Calibration

The current route-state vector is:

```text
Z(V,t) = [B_in, B_out, Q_coh, J_bound, J_trans, D, I_comp, L_diff, X_frag]
```

All of these remain useful, but not all require equal attention in the next phase.

The highest-priority variables are:

- `Q_coh`
- `J_trans`
- `B_out`
- `I_comp`
- `L_diff`
- `X_frag`

The reason is simple.

`B_in` and `J_bound` are already comparatively intelligible in many cases. They matter, but they are not yet the main uncertainty.

The main uncertainty is concentrated in the variables that govern:

- throughput pressure;
- coupling quality;
- competition and interference;
- drift versus break;
- the onset of fragmentation.

Those are the variables that most clearly decide whether a machine remains in `T0`, moves into `T1`, degrades into `T2`, crosses into `T3`, or breaks into `T4`.

## Variable Priorities

## 1. `Q_coh`

`Q_coh` is the single most important calibration target because it carries too much weight in the current corpus to remain only verbally specified.

At machine level, `Q_coh` needs to be read as:

```text
the degree to which the machine's parts remain coupled strongly enough, in the right order, with the right selectivity, to preserve the machine's characteristic route identity under load
```

The next phase should therefore sharpen `Q_coh` by splitting its practical reading into:

- structural coherence;
- sequence or recognition fidelity;
- timing or cycle coherence;
- boundary or environment coherence;
- product or output coherence.

These may still roll back into one high-level variable, but they should first be inspected separately.

## 2. `J_trans`

`J_trans` is the second major calibration target because it expresses route intensity, but intensity is not self-interpreting.

High `J_trans` can mean:

- productive throughput;
- dangerous pressure;
- unstable overspeed;
- compensatory overdrive;
- poorly governed escalation.

The next phase should therefore stop treating `J_trans` as merely:

```text
how much transition is occurring
```

and instead read it as:

```text
how strongly route-bearing transition is being driven relative to the machine's present coherence and boundary capacity
```

In practice, `J_trans` must be calibrated alongside `Q_coh`, not alone.

## 3. `B_out`

`B_out` matters because it is where machine work becomes machine strain.

The next phase should treat `B_out` as a structured load variable, not a generic exterior pressure term.

At machine level that load often comes from:

- gradient demand;
- substrate scarcity;
- product backlog;
- oxidative pressure;
- repair burden;
- turnover demand.

That means `B_out` should be decomposed by source during calibration work even if it later remains collapsed in the formal summary.

## 4. `I_comp`

`I_comp` is currently useful but under-disciplined.

It should no longer mean only:

```text
interference exists
```

It should instead be read as:

```text
the degree to which rival routes, side reactions, noise, or occupancy conflicts are strong enough to reduce admissible machine order
```

This is especially important because `I_comp` is one of the main bridges between:

- healthy regulation;
- manageable competition;
- pathological side-routing;
- corruption-like decoupling at machine scale.

## 5. `L_diff`

`L_diff` needs stronger calibration because it is one of the main signatures of non-catastrophic failure.

It often marks:

- leak without immediate break;
- reduced selectivity;
- reduced confinement;
- wasted energy or order;
- burden accumulation before crisis.

That makes it critical for distinguishing:

- `T2` metastable drift

from

- `T4` fragmentation.

If this variable remains too loose, the corpus will keep collapsing slow degradation into the language of collapse.

## 6. `X_frag`

`X_frag` is the most obviously terminal variable, but it still needs calibration because fragmentation is not binary in practice.

There are at least three layers that should be separated:

- local fragment production;
- propagating fragmentation burden;
- threshold-level identity break.

The next phase should therefore avoid treating every rise in `X_frag` as equal.

The question is not merely:

```text
is fragmentation present?
```

The question is:

```text
has fragmentation become strong enough, persistent enough, and system-owning enough to abolish the former route identity?
```

## Threshold Relations That Matter Most

The next phase should focus on threshold relations rather than isolated variables.

The most important threshold relations are the following.

## 1. Coherence-to-Transition Threshold

The most important relation in the whole phase is:

```text
Q_coh relative to J_trans
```

If `J_trans` rises while `Q_coh` remains strong, the machine may intensify productively.

If `J_trans` rises while `Q_coh` falls, the machine may enter:

- overdriven drift;
- side-routing;
- fragmentation risk.

This is the main threshold relation for distinguishing:

- strong throughput

from

- destructive pressure.

## 2. Load-to-Retention Threshold

The next important relation is:

```text
B_out relative to B_in
```

This matters because machines do not fail only when activity ceases. They fail when outward demand persistently outruns retained order.

This relation is especially important for:

- membrane-dependent energy systems;
- translation under renewal burden;
- repair-limited stressed cells.

## 3. Competition-to-Admissibility Threshold

The third major relation is:

```text
I_comp relative to route admissibility
```

The practical question here is:

```text
when do side routes, occupancy conflicts, noise, or branch competition stop being governable and begin changing the machine's effective route class?
```

This is one of the strongest remaining discriminants between:

- healthy complexity;
- stressed compensation;
- decoupled dysfunction.

## 4. Diffusion-to-Fragmentation Threshold

The fourth key relation is:

```text
L_diff relative to X_frag
```

This threshold matters because some systems lose efficiency long before they lose identity.

The next phase should sharpen the difference between:

- leakage that still permits recovery;
- leakage that forces metastable compensation;
- leakage that accelerates fragmentation into irreversible break.

This is a major control point for keeping the ontology honest about continuity.

## 5. Machine-to-Cell Threshold

The fifth key relation is:

```text
local machine failure relative to whole-cell state transition
```

A machine can weaken without killing the cell.
A machine can compensate while the cell remains viable.
A machine can fail in a way that propagates fast enough to move the cell from stress compensation into collapse.

This threshold relation matters because the corpus now has a:

```text
machine -> life -> person
```

ladder, and that ladder will remain too loose unless machine failure can be related more tightly to cell-state transitions.

## Best Machine Cases for Constraint Gain

Not all cases are equally useful.

The strongest cases for this phase are the ones where:

- variables are already partially mapped;
- failure modes are comparatively clear;
- family identities are densely loaded into the system;
- threshold shifts are biologically meaningful.

That gives the following order.

## 1. Respiration / ATP Synthesis

This remains the best first calibration case.

Why:

- it loads `Fe`, `S`, `O`, and `P` strongly;
- it has clear gradient, leak, closure, and fragmentation logic;
- failure propagation is already mapped;
- it transitions clearly from productive order into oxidative stress and collapse.

This case is best for calibrating:

- `Q_coh`
- `J_trans`
- `B_out`
- `L_diff`
- `X_frag`

It is the strongest current pressure chamber for threshold work.

## 2. Ribosome / Translation

This remains the best second case.

Why:

- it loads `N`, `P`, `O`, and `C` densely;
- it distinguishes fidelity drift from catastrophic break better than respiration does;
- it gives a clearer picture of metastability and burden accumulation;
- it exposes how defect propagation can remain recoverable for a long interval before collapse.

This case is best for calibrating:

- `Q_coh`
- `I_comp`
- `L_diff`
- product-burden coupling;
- metastable drift signatures.

It is the best current case for refining `T2`.

## 3. Redox Repair / Antioxidant Governance

This is the best third case because it sits at the edge between compensation and failure.

Why:

- it links machine function to repair burden;
- it clarifies how a system remains coherent under damage pressure;
- it distinguishes stress compensation from terminal fragmentation;
- it tightens the machine-to-cell threshold.

This case is best for calibrating:

- compensatory `J_trans`;
- repair-sensitive `B_out`;
- rising `X_frag` under partial containment;
- `T2` to `T3` threshold movement.

## 4. Boundary and Membrane Maintenance

This is a strong fourth case because many high-level failures become lethal only when boundary coherence is lost.

Why:

- it couples energy, transport, retention, and environmental control;
- it exposes the role of `B_in` without reducing the phase to `B_in` alone;
- it clarifies how leak becomes identity threat.

This case is best for calibrating:

- `B_in`
- `B_out`
- `L_diff`
- local-to-global propagation.

## Working Threshold Hypotheses

The next phase should treat the following as working hypotheses to be tested and refined.

## Hypothesis 1

```text
Machines usually enter T2 when Q_coh falls faster than J_trans falls.
```

That is, throughput may continue while order quality weakens.

## Hypothesis 2

```text
Machines usually enter T3 when B_out persistently outruns the machine's effective retention and interface capacity.
```

This frames threshold crossing as sustained mismatch, not momentary disturbance.

## Hypothesis 3

```text
High I_comp becomes dangerous primarily when it changes route admissibility rather than merely lowering efficiency.
```

That means competition becomes ontologically serious when it rewrites the route, not merely when it slows it.

## Hypothesis 4

```text
L_diff is the main early warning variable for recoverable decline, while X_frag is the main late-stage variable for identity break.
```

This is a strong working distinction that should now be pressure-tested.

## Hypothesis 5

```text
Whole-cell state shifts occur when multiple machine-level threshold crossings become mutually reinforcing rather than isolated.
```

This is essential for preventing false one-machine reductionism.

## What This Phase Should Avoid

Avoid:

- new broad synthesis before calibration gains;
- premature extension into social or symbolic domains;
- pretending approximate values are precise measurements;
- treating every perturbation as fragmentation;
- multiplying variables faster than they can be constrained.

The purpose of the phase is to increase discrimination, not to increase vocabulary.
