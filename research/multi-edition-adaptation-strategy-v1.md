## Purpose

This document defines how `Coherent Biblical Ontology` should be turned into multiple editions for different reading and cognition levels without allowing ontology drift, doctrinal drift, or uncontrolled rewriting.

The current canonical English master is:
- 15 chapters
- 4 appendices
- about 34.8k words

Earlier audience research was done against a broader AMS v2.0 planning horizon, including a possible 25-chapter master spine. That older planning work is still useful for audience behavior, but the adaptation programme must now map to the actual current 15-chapter master, not to a superseded outline.

## Core Principle

There must be:
- one canonical ontology source
- multiple derived reading editions
- no independent parallel authorship of the same book

This means:
- the master book defines the ontology
- derived editions adapt delivery, not doctrine
- every derived edition must be traceable back to the canonical source

## Canonical And Derived Layers

### Canonical Layer

The canonical layer is the ontological authority for the project:
- chapter order
- core claims
- scripture anchors
- ontology stack
- baseline terminology
- corruption taxonomy
- redemption structure
- appendix logic

The canonical source must not be simplified directly into lower-level editions.

### Derived Layers

Derived editions exist for audience fit:
- GCSE / general intelligent reader
- A-level / strong sixth-form
- Bachelor’s / educated non-specialist
- PhD / professional reference

These editions may differ in:
- sentence length
- abstraction density
- recap frequency
- example count
- glossary load
- appendix placement
- chapter subdivision

They must not differ in:
- the ontology itself
- the theological frame
- the status of baseline claims
- the distinction between ontology and mechanism

## Why Bachelor’s Should Come First

The first derived edition should be the Bachelor’s version.

Reasons:
- it is the best midpoint between accessibility and precision
- it is close enough to the master to preserve nuance
- it is far enough from the master to force real editorial discipline
- it provides the best source for downward adaptation into A-level and GCSE
- it provides the best control version for upward adaptation into a denser PhD/professional edition

If GCSE is built first, too much structure will be flattened too early.
If PhD is built first, too little will be learned about adaptation pressure.

## Audience Ladder

### GCSE

Needs:
- short conceptual steps
- concrete-first exposition
- frequent recap
- low jargon tolerance
- explicit transitions

Main risk:
- overload and discouragement

### A-level

Needs:
- real abstraction, but signposted
- explicit layer tracking
- slower transitions than the master
- more bridge paragraphs than the master

Main risk:
- layer confusion

### Bachelor’s

Needs:
- visible hierarchy of claims
- disciplined terminology
- medium-to-long chapters
- examples used for testing rather than decoration
- clear distinction between baseline ontology and speculative mechanism

Main risk:
- hidden dependency chains

### PhD / Professional

Needs:
- strict status precision
- explicit inferential architecture
- stronger objection handling
- full mechanism-status tagging

Main risk:
- rhetorical inflation

## Non-Negotiable Preservation Rules

The following must remain stable across all editions:
- Scripture-first frame
- creation/runtime distinction
- continuity before discreteness
- configuration before assembly
- ontology stack logic
- cross-cutting role of `bara`, `asah`, `yatsar`
- corruption as parasitic disorder, not rival ontology
- redemption as renewal of real creaturely identity
- local assembly framing where church is named
- AI as derivative and appendical in the current master line

## What May Be Adapted

The following may be adapted by edition:
- paragraph length
- number of examples
- explicitness of transitions
- chapter internal subsectioning
- glossary depth
- amount of recap
- whether appendix material stays in the back or is pulled into sidebars/notes
- amount of mechanism detail kept in the main line

## What Must Never Be Smoothed Away

The following simplifications would be harmful:
- replacing AMS-specific terms with mainstream approximations just to sound easier
- collapsing ontology and mechanism into one blurred layer
- removing theological guardrails rather than explaining them
- flattening scripture into decorative quotation
- softening the claim-status difference between ontology, mechanism, and speculation

## Edition Architecture Rules

### Rule 1: One Source, Many Views

Every edition must be derived from the same canonical chapter map.

### Rule 2: Chapter Mapping Stays Stable

Even when lower-level editions split chapters internally, they should still preserve traceability back to the canonical 15-chapter structure.

### Rule 3: Claims Must Be Typed

Each edition should preserve, in proportion to its audience:
- baseline ontological claim
- explanatory expansion
- mechanism candidate
- theological warrant
- implication or application

The amount of explicit marking can vary, but the distinction must remain.

### Rule 4: Lower Editions Need More Bridges, Not Less Ontology

Downward adaptation should add:
- orienting sentences
- chapter stakes
- term definitions
- recap-ready endings

It should not simply delete difficult thought and hope the result is clearer.

### Rule 5: Higher Editions Need More Status Discipline, Not More Noise

Upward adaptation should add:
- objection handling
- status markers
- sharper distinction between ontology and mechanism
- more explicit comparative framing

It should not merely inflate vocabulary.

## Practical Production Model

The cleanest production model is:

1. freeze the current canonical master
2. define edition rules
3. build one Bachelor’s workspace
4. pilot a single chapter
5. assess adaptation pressure
6. adapt the whole book
7. derive A-level from the Bachelor’s line
8. derive GCSE from the A-level or Bachelor’s line
9. derive PhD/reference from the canonical master
