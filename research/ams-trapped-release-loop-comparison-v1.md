# AMS Trapped Release Loop Comparison v1

## Purpose

This document compares the current loop geometries across the three most important stall contexts in the programme:

- glass exit
- crystal exit
- glass-to-crystal compatibility stall

The goal is to determine:

- which loop families are specifically release-side
- which loop families appear to be more general interface-stall geometries
- whether the same spatial logic recurs across exit and compatibility failures

This matters because the programme is now far enough along that it should begin distinguishing:

- local geometries that belong to one narrow case
- local geometries that reflect a deeper recurring AMS interface pattern

## Loop Families in Scope

From:

- `ams-terminal-layer-depth-and-loop-geometry-v1.md`

the active loop classes are:

- `L1` face loop
- `L2` shallow pocket loop
- `L3` elongated stall loop
- `L4` broken multi-loop field

## Loop-by-Loop Analysis

## 1. `L1` Face Loop

### Geometry

- very low penetration depth
- strongly surface-concentrated
- little through-layer progression

### In Glass Exit

This remains plausible, especially when the terminal face is active enough to form a local event but not structured enough to sustain deeper continuation.

Best fit:

- shallow trapped release
- rough or weakly contaminated surface
- immediate near-face reseating

### In Crystal Exit

This is less central.

Crystal exit failures appear more likely to involve:

- structured return
- or deeper conversion failure

than simple face-only looping.

So `L1` is possible, but not the strongest crystal-exit failure geometry.

### In Glass->Crystal Stall

This is the weakest fit of the three contexts.

The `glass -> crystal` problem usually appears to require a finite transition layer with real mediation burden. A face-only loop looks too shallow for the typical compatibility-stall problem.

### Current Judgment

`L1` is mainly a release-side geometry, especially on the glass side.

It is not the strongest candidate for deeper compatibility failures.

## 2. `L2` Shallow Pocket Loop

### Geometry

- low/medium penetration depth
- bounded local interface volume
- stronger local retention than a face loop

### In Glass Exit

This is one of the strongest current glass exit loop candidates.

Why:

- the event can enter a bounded local release volume
- outward continuation can fail without immediate incoherence
- the glass whole does not require highly patterned structuring for this to happen

### In Crystal Exit

This is plausible, but not dominant.

Crystal exits seem more likely to push either toward:

- cleaner release
- structured return
- or more ordered elongated stall

rather than pocket-like local retention.

Still, local pocket trapping near a selectively releasing surface remains possible.

### In Glass->Crystal Stall

This is plausible as an early-stage compatibility stall.

The event may enter a bounded compatibility region, but fail to progress into full crystal-style admissibility.

However, the stronger compatibility-stall cases seem to go beyond shallow pockets.

### Current Judgment

`L2` is strong in glass release, plausible in compatibility stall, and secondary in crystal exit.

## 3. `L3` Elongated Stall Loop

### Geometry

- medium penetration depth
- non-trivial progression through the terminal or interface layer
- failure occurs after partial advancement, not at first contact

### In Glass Exit

This is a strong candidate for the more mature trapped-release cases.

Why:

- it fits terminal-layer stall rather than face-only retention
- it explains why glass can transmit well in bulk but still fail in a deeper release zone

### In Crystal Exit

This is also strong.

Why:

- crystal release failures are more likely to involve partially successful conversion of a strongly patterned internal event
- such a case naturally fits an elongated progression followed by failure

### In Glass->Crystal Stall

This is the strongest current loop for the compatibility-stall case.

Why:

- the problem is explicitly one of partial transition between transmissive grammars
- the event needs a real transition zone
- shallow face-only retention does not do enough explanatory work

This makes `L3` the clearest recurring cross-context stall geometry in the entire current programme.

### Current Judgment

`L3` is not release-specific.

It appears to be a general interface-stall geometry that recurs across:

- glass release
- crystal release
- glass-to-crystal compatibility failure

That is one of the strongest results in the document.

## 4. `L4` Broken Multi-Loop Field

### Geometry

- fragmented local reseating
- weak loop coherence
- movement toward scattering

### In Glass Exit

This is the best boundary case between:

- trapped release
- and scattering at exit

### In Crystal Exit

This is less central, because crystal-like failures tend to preserve either:

- stronger order
- or stronger return tendencies

before degrading into fully broken loop fields.

### In Glass->Crystal Stall

This is again a boundary case.

If compatibility failure becomes too fragmented, it ceases to look like genuine stall and starts to look like:

- broken mediation
- localised incoherence
- or scattered failure

### Current Judgment

`L4` is less a primary loop family than a transition-to-breakdown regime.

It is useful, but mainly as the edge where bounded stall ceases to dominate.

## Cross-Context Findings

## 1. Some Loops Are Release-Specific, Some Are General

Current best distinction:

- `L1` is mainly release-side and especially surface-concentrated
- `L2` is strongest in glass release and near-release compatibility pockets
- `L3` is the strongest general interface-stall geometry
- `L4` is a breakdown-edge pattern rather than a primary stable family

## 2. `L3` Is the Most Important Recurring Stall Family

This is the main result.

`L3` appears to unify:

- glass terminal-layer stall
- crystal release conversion stall
- glass-to-crystal compatibility stall

That means the programme now has at least one loop family that looks genuinely cross-context rather than ad hoc.

## 3. Release and Compatibility Failures Are Not Identical

The loop comparison does not collapse everything into one stall class.

Important distinctions remain:

- glass release is more hospitable to `L1` and `L2`
- crystal exit is more selective and more compatible with ordered return or `L3`
- glass->crystal stall is most strongly `L3` and only secondarily `L2`

So there is genuine family resemblance without identity.

## 4. The Transition Layer Is Now Clearly Doing Real Work

The loop comparison strongly supports the claim that:

- finite interface regions
- with real spatial depth
- and recurring local loop patterns

are doing explanatory work in the AMS programme.

This is no longer just a metaphorical way of speaking about interfaces.

## Strategic Consequences

## 1. The Programme Now Has a Recurring Stall Geometry

That is important because it means the next steps can deepen one promising geometric family rather than invent a new story for every case.

That recurring family is:

- `L3` elongated stall loop

Because it carries:

- `L1`

## 3. Glass->Crystal Remains the Best Compatibility Frontier

Because it gives the clearest non-release-specific stress test of:

## 4. Crystal Exit Should Be Treated as the Ordered-Release Bridge

It now looks less like the first exit case to solve, and more like the case that bridges:

- release
- ordered conversion
- and return

once the broader release logic is more stable.
