# AMS Cell Failure Thresholds and Death Modes v1

## Purpose

This note maps whole-cell failure modes after the cell-state synthesis pass.

The preceding synthesis proposed:

```text
A viable cell is a carbon-built, nitrogen-readable, phosphorus-ordered, sulfur-governed, iron-powered, oxygen-conditioned runtime loop.
```

That makes the next question unavoidable:

```text
how does that loop actually fail?
```

This file treats cell death and collapse as threshold crossings in coupled runtime coherence rather than as a flat list of named outcomes.

## Central Claim

The central claim is:

> Cell death modes are not arbitrary endings imposed from outside the runtime grammar. They are distinct failure topologies in which architecture, readability, transferable order, reactive governance, redox depth, and polarity/oxidative field cease to remain mutually supportable. Different death modes are different decoupling orders.

## The Six Failure Axes

The cell-state synthesis implies six major axes of collapse.

### 1. Architectural Failure

Dominant family:

```text
carbon
```

Question:

```text
can the cell still maintain host form, compartment persistence, and machine-bearing structure?
```

Primary failure features:

- membrane rupture or uncontrolled loss of integrity;
- cytoskeletal or compartment collapse;
- severe proteostatic deformation;
- loss of boundary persistence.

### 2. Readability Failure

Dominant family:

```text
nitrogen
```

Question:

```text
can the cell still read, match, and act on encoded biological order?
```

Primary failure features:

- translation collapse;
- recognition failure;
- coding fidelity breakdown;
- informational lesions outrunning recovery.

### 3. Transfer Failure

Dominant family:

```text
phosphorus
```

Question:

```text
can the cell still capture and move ordered state through ATP, phosphorylation, and charged backbone continuity?
```

Primary failure features:

- ATP depletion;
- phosphate-cycle arrest;
- signalling permission failure;
- loss of directional runtime progression.

### 4. Governance Failure

Dominant family:

```text
sulfur
```

Question:

```text
can dangerous chemistry still be buffered, repaired, and kept in bounded routes?
```

Primary failure features:

- thiol-buffer collapse;
- disulfide mismanagement;
- repair mesh overload;
- runaway chemically dangerous states.

### 5. Redox-Depth Failure

Dominant family:

```text
iron
```

Question:

```text
is deep electron-state capacity still embedded and governed, or has it become free destructive pressure?
```

Primary failure features:

- labile iron expansion;
- Fe-S collapse;
- catalytic depth escaping containment;
- depth becoming toxicity.

### 6. Oxidative-Field Failure

Dominant family:

```text
oxygen
```

Question:

```text
is oxygen power still field-conditioned and productive, or has it fragmented into membrane, protein, and informational damage?
```

Primary failure features:

- ROS escalation;
- proton-field collapse;
- lipid peroxidation;
- oxidative damage outrunning repair.

## Failure Topology Classes

The current system supports five major death-mode classes.

## 1. Ordered Shutdown Failure

This is the cleanest failure topology.

AMS route:

```text
damage or stress recognition
-> controlled permission changes
-> organised machine disengagement
-> bounded dismantling
```

This is the most orderly collapse mode. It does not mean "good" or "harmless." It means coherence is lost through a staged shutdown rather than a diffuse spill.

Dominant family reading:

```text
N/P/C/O with governance support
```

Interpretation:

- nitrogen-readable state change marks that continuation is no longer permitted;
- phosphorus-linked permissions and cycle-state controls shift;
- carbon architecture is dismantled in bounded sequence;
- oxygen remains present but not yet in dominant runaway fragmentation mode.

Vorton interpretation:

```text
ordered-shutdown vorton = staged decoupling of runtime permissions under retained boundary control
```

Route profile:

```text
route class: R4/R5
obstruction class: O2/O4
dominance: D4
grade: G_B/G_C
```

Mechanism-facing statement:

```text
Ordered shutdown is cell failure with route discipline retained longer than viability.
```

## 2. Necrotic Energy Failure

This is the classic energy-collapse mode.

AMS route:

```text
respiration failure or extreme metabolic insult
-> ATP collapse
-> transport failure
-> ion and water imbalance
-> membrane rupture and architectural loss
```

Dominant family reading:

```text
Fe/O/P failure propagating into C-boundary collapse
```

Interpretation:

- iron depth no longer produces useful order;
- oxygen closure is absent, insufficient, or dysfunctional;
- phosphorus transfer capacity falls below viability threshold;
- carbon boundary architecture loses host integrity.

Vorton interpretation:

```text
necrotic-collapse vorton = loss of transferable order followed by host rupture
```

Route profile:

```text
route class: R3/R6
obstruction class: O2/O5/O6
dominance: D3
grade: G_D
```

Mechanism-facing statement:

```text
Necrotic failure begins as ATP-order collapse and ends as carbon-boundary disintegration.
```

## 3. Ferroptotic Membrane Failure

This is one of the clearest threshold-defined collapse modes in the current model.

AMS route:

```text
iron pressure + lipid peroxide propagation + sulfur-governance exhaustion
-> membrane boundary collapse
```

Dominant family reading:

```text
Fe/O/S collapse expressed through C/P/O boundary damage
```

Interpretation:

- iron depth accelerates damage;
- oxygen fragmentation drives lipid peroxide propagation;
- sulfur-mediated repair fails to keep pace;
- the carbon-built membrane host loses persistence.

Vorton interpretation:

```text
ferroptotic-failure vorton = iron-amplified oxygen fragmentation overrunning membrane repair
```

Route profile:

```text
route class: R6
obstruction class: O5/O6
dominance: D3
grade: G_D
```

Mechanism-facing statement:

```text
Ferroptosis is boundary collapse driven by iron-depth escape and oxygen-fragmentation propagation after sulfur governance fails.
```

## 4. Translation-Collapse Failure

This is the main information-to-architecture failure mode.

AMS route:

```text
readability loss
or charging/cycle failure
or sustained mistranslation burden
-> defective machine renewal
-> proteostatic overload
-> architecture and energy systems fail secondarily
```

Dominant family reading:

```text
N/P/C collapse with O-conditioned field stress
```

Interpretation:

- nitrogen readability no longer remains reliable;
- phosphorus cycle order no longer supports coherent translation;
- carbon product architecture becomes defective or insufficiently renewed;
- oxygen-conditioned aqueous order can no longer compensate for rising misfolding and burden.

Vorton interpretation:

```text
translation-collapse vorton = decoupling of readability, delivery, and architectural renewal
```

Route profile:

```text
route class: R3/R5/R6
obstruction class: O2/O5
dominance: D3/D4
grade: G_C/G_D
```

Mechanism-facing statement:

```text
Translation collapse kills the cell by starving it of correct self-renewal before energy necessarily fails first.
```

## 5. Oxidative-Informational Hybrid Failure

Some failure states are not purely membrane-first or energy-first. They are mixed states in which oxidative damage and informational damage couple.

AMS route:

```text
ROS escalation
-> protein and nucleic-acid damage
-> translation and repair fidelity loss
-> further ROS and energy disorder
-> accelerating whole-cell collapse
```

Dominant family reading:

```text
O/S/Fe failure feeding into N/P/C degradation
```

Interpretation:

- oxygen fragmentation outruns sulfur repair;
- iron may amplify the damage;
- nitrogen-rich readable systems and phosphorus-backed continuity are injured;
- carbon architecture loses correct renewal and eventually host integrity.

Vorton interpretation:

```text
oxidative-informational vorton = feedback loop between oxygen damage and readability loss
```

Route profile:

```text
route class: R6
obstruction class: O5/O6
dominance: D3
grade: G_D
```

Mechanism-facing statement:

```text
Hybrid failure occurs when oxygen damage stops being merely chemical and begins eroding the cell's own ability to interpret and repair itself.
```

## Threshold Logic

These death modes differ not only by symptoms but by threshold order.

### Threshold Order A: Energy First

```text
Fe/O/P axis fails first
-> transport and renewal weaken
-> boundary and readability collapse later
```

### Threshold Order B: Boundary First

```text
C/P/O host integrity fails first
-> gradients and routing dissolve
-> energy and readability collapse secondarily
```

### Threshold Order C: Repair First

```text
S governance fails first
-> O fragmentation rises
-> Fe depth becomes more dangerous
-> architecture and information collapse later
```

### Threshold Order D: Readability First

```text
N/P cycle coherence fails first
-> machine renewal degrades
-> energy systems and boundary systems fail secondarily
```

This is the main reason the death-mode taxonomy matters. It preserves causal order instead of collapsing all failures into one terminal state.

## Compact Death Grammar

The best current compression is:

```text
Cells die when one or more core runtime axes cease to remain coupled strongly enough for renewal, repair, and bounded field order to continue.
```

More explicitly:

```text
Death is not the mere absence of life processes.
It is the threshold at which architecture, readability, transferable order, reactive governance, redox depth, and oxygen-conditioned field order can no longer hold each other in a self-sustaining loop.
```
