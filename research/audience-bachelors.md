# Audience Profile: Bachelor’s Degree Readers (age ~21-22)

Date: 2026-03-11
Status: research baseline for AMS v2.0 audience review

## Purpose

This document defines the reading and conceptual profile for a bachelor’s-level AMS reader, especially one comfortable with undergraduate physics, philosophy, theology, or adjacent analytic reading.

## Research Basis

Online reference points used:
- OpenStax University Physics learning design: conceptual exposition, examples, chapter summaries, key terms, and layered derivations.
- University writing guidance for academic/scientific prose: clarity, structure, audience awareness, and explicit argument control.
- Philosophy-writing guidance emphasising clear thesis statements, premise tracking, and objection handling.

Research takeaway:
- this readership can tolerate abstraction if the structure is visible
- precision matters more than friendliness
- repetition should be strategic, not constant
- examples still matter, but not every idea needs to begin with one

## Reader Baseline

Likely safe assumptions:
- comfort with formal argument
- comfort with model critique
- ability to track layered distinctions
- willingness to read slower for difficult material

Unsafe assumptions:
- agreement with AMS premises
- theological sympathy
- patience for vague rhetoric or repeated throat-clearing

## Sentence Structure

Recommended style:
- medium to long sentences are acceptable
- clause density can be moderate to high if syntax stays controlled
- premise-conclusion movement can be explicit without overnarration

Tolerance profile:
- nested reasoning: moderate to high
- abstraction: high if defined
- compressed synthesis: moderate

## Vocabulary Profile

This audience can handle stable use of:
- ontology
- epistemology
- mechanism
- admissibility
- configuration
- coherence
- derivation
- anti-reification

Still required:
- term discipline
- first-use clarity
- avoidance of unnecessary synonym drift

## Explanatory Method

Best fit:
- definition, argument, application
- chapter claims stated early
- examples used to test claims rather than merely decorate them

This audience benefits from:
- clear inferential architecture
- section-end implications
- explicit contrast with rival descriptions

## Assumption Load

This audience can generally track:
- 4 fresh premises in one chapter if the hierarchy is visible
- 6+ active premises if some are already established and lightly restated

Failure risk:
- when premises are present but their dependency order is hidden

## Axiom Consumption Capacity

Recommended ceiling:
- 4 new axioms or major distinctions in a chapter
- but only if one is clearly primary and the others are subordinate

Recommended handling:
- formal premise clusters are acceptable
- chapter architecture should show which claims are baseline, which are provisional, and which are speculative
- examples should test, not replace, conceptual rigor

## Conceptual Pacing

Recommended pace:
- brisk but not rushed
- fewer recaps than A-level
- more synthesis between chapters

Useful supports:
- explicit chapter stakes
- “what depends on this chapter” statements
- glossary only for the most framework-specific terms

## Tone

Target tone:
- mature
- analytically direct
- calm
- unshowy
- conceptually responsible

Avoid:
- classroom over-scaffolding
- motivational padding
- rhetorical overclaim

## Structural Expectations

Best fit:
- substantial chapters with clear subsections
- occasional diagrams/tables
- chapter-end summary or “what this establishes”
- distinction between canonical claims and mechanism candidates

## Theological Handling

This audience can track:
- theological authority claims
- doctrinal ceilings
- level-tagged interactions between doctrine and ontology

But the prose must:
- distinguish theological warrant from physical inference
- explain why some claims are guarded rather than merely asserted

## Reader Fatigue Risk

Likely overload points:
- if every chapter front-loads caveats before making progress
- if theological, ontological, and physical categories are mixed without headings
- if the same concept is renamed too often

## Common Failure Modes

1. Overexplaining basic structure and underexplaining difficult distinctions.
2. Using too much conceptual compression too early.
3. Hiding uncertainty status.
4. Treating speculative mechanism as though its epistemic status is obvious.

## Writing Guidance For This Audience

The future bachelor’s version should:
- tighten prose
- preserve hierarchy of claims
- reduce repetition
- retain examples only where they test comprehension or reveal explanatory gain

## Recommended Master-Manuscript Implications

The master book should preserve:
- visible claim hierarchy
- explicit distinction between baseline ontology and speculative mechanism
- cross-chapter dependency logic
- disciplined glossary control
