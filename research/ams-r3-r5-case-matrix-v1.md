# AMS R3/R5 Case Matrix v1

## Purpose

This document compares the two main frontier route classes directly across the strongest benchmark cases:

- `R3` blocked apparent route
- `R5` loop-dominated pseudo-route

Benchmark cases:

- `glass -> air`
- `glass -> crystal`
- `crystal -> air`

The purpose is to stop relying on separate descriptive accounts and instead force a direct side-by-side judgment of:

- route reality
- loop dominance
- stage of failure
- true versus apparent `L_escape`

## Core Distinction Being Tested

Current best distinction:

- `R3` is a failed real route
- `R5` is a pseudo-route created by dominant local looping

This matrix tests whether that distinction survives close comparison under the same benchmark conditions.

## Comparison Criteria

### Route Reality

Does a genuinely completion-directed route exist at all?

### Loop Dominance

Is looping secondary to failed progression, or is looping the main event?

### Stage of Failure

Does failure occur:

- after real progression
- during partial transition
- or before any genuine completion route ever forms?

### `L_escape`

Is completion-side viability:

- genuinely present but weak
- or mostly only apparent because local activity is intense?

## Case-by-Case Analysis

## 1. Glass -> Air

### `R3` in Glass Release

This is the case where:

- outward release genuinely begins
- a completion route appears real
- the terminal layer carries the event partway
- the route breaks under actual continuation

This fits:

- blocked terminal continuity
- release that seems nearly viable
- late failure inside the release zone

### `R5` in Glass Release

This is the case where:

- local release activity is strong
- reseating near the terminal layer is real
- but the event never possesses a genuinely completion-directed route

The activity pattern imitates release without becoming release.

### Main Matrix Judgment

This is the strongest release-side benchmark for the distinction.

The key difference is:

- `R3`: release is genuinely underway, then blocked
- `R5`: release-like behaviour is mostly local cycling from the start

### Best Quick Test

Ask:

- did the event ever have a route that could really have completed outwardly?

If yes:

- likely `R3`

If no:

- likely `R5`

## 2. Glass -> Crystal

### `R3` in Compatibility Stall

This is the case where:

- narrowing toward crystal-style admissibility begins
- the transition layer supports real partial mediation
- final handoff fails because the route is blocked or never fully receivable

This is one of the strongest examples of:

- genuine but unsuccessful completion-side route formation

### `R5` in Compatibility Stall

This is the case where:

- the transition zone becomes active
- local narrowing-like behaviour appears
- the event seems to be progressing toward admissibility
- but the activity is mainly local looping or pseudo-progression

No true admissible route ever stabilises.

### Main Matrix Judgment

This is the strongest compatibility-side benchmark for the distinction.

It is also one of the most valuable cases in the whole programme because both classes can look very similar from a distance.

### Best Quick Test

Ask:

- was admissible handoff ever becoming genuinely real?

If yes, then blocked late:

- `R3`

If no, despite active local transition behaviour:

- `R5`

## 3. Crystal -> Air

### `R3` in Crystal Release

This is the cleaner case.

The event:

- is already more ordered internally
- begins a structured release conversion
- fails before completion

This fits `R3` well because crystal release tends to preserve genuine route reality when it fails.

### `R5` in Crystal Release

This is weaker.

Local looping can happen, but it is not the dominant frontier mode here because crystal exit failures more often appear to move toward:

- narrow viable route
- ordered return
- or blocked ordered release

rather than sustained pseudo-route activity.

### Main Matrix Judgment

This case is useful mostly as a contrast case.

It shows that:

- not every benchmark gives `R3` and `R5` equal weight

and that itself is a useful finding.

## Strongest Comparative Conclusions

## 1. `R3` and `R5` Are Both Real in Glass Cases

This is the strongest result in the matrix.

Both:

- `glass -> air`
- `glass -> crystal`

are genuine mixed test beds where:

- route blockage
- and pseudo-route looping

can both plausibly appear.

That is why glass remains the best frontier material in the programme.

## 2. `R3` Dominates More Clearly in Crystal Release

This is also important.

It suggests that more ordered source wholes are less hospitable to sustained pseudo-route behaviour and more likely to preserve:

- genuine but fragile route structure

even when they fail.

## 3. The Best Discriminator Is Still Route Reality

Across all three benchmarks, the strongest question remains:

- did a genuinely completion-directed route ever exist?

Everything else helps, but this is still the best first discriminator.

## 4. Loop Dominance Is the Second-Best Discriminator

If looping is:

- secondary to real progression, think `R3`
- primary and self-maintaining, think `R5`

## 5. `L_escape` Must Be Read Carefully

This matrix strongly confirms:

- true `L_escape` is not the same as visible activity

That is now one of the most stable conclusions in the whole recent modelling sequence.

## Practical Use Rule

When diagnosing a stalled event:

1. ask whether a real completion route existed
2. ask whether looping was primary or secondary
3. ask whether failure came after genuine progression or before one route ever stabilised
4. ask whether `L_escape` is real or merely inferred from activity

If the answers are:

- real route
- looping secondary
- later failure
- weak but real `L_escape`

then the event is likely:

- `R3`

If the answers are:

- no stable real route
- looping primary
- pseudo-progressive activity
- apparent but not real `L_escape`

then the event is likely:

- `R5`

## Programme-Level Importance

This is the main result.

The route-class split between:

- failed real route

survives direct comparison under the same benchmarks.

## 2. Glass Remains the Best Frontier Material

## 3. Crystal Provides the Useful Contrast Case

Crystal is less fertile for `R5`, but that is useful because it shows the distinction is not being imposed everywhere indiscriminately.
