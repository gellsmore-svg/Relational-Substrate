# AMS Substrate Coherence Research Framework v1

## Purpose

This document defines the next technical research programme for AMS.

The core claim is not that `T0` can be proved by direct observation as a standalone object. The stronger claim is narrower:

- `T0` may be inferred as the most coherent ontological basis for the behaviours, structures, and persistence patterns observed across chemistry and higher-order formations

That means the technical task is not to search for a single decisive experiment that directly “shows the substrate.” The task is to build a disciplined coherence-testing framework and then see whether AMS can carry real structure better than thinner alternatives.

## Research Goal

Build a `substrate-coherence test framework` that asks whether AMS can map:

- stable chemical structure
- unstable or transient topology classes
- higher-order formations

without collapsing into ad hoc explanation.

The framework should make it possible to compare:

- observed chemistry and material behaviour
- AMS ontological interpretation
- alternative thinner interpretations

## Governing Principle

AMS should gain strength by surviving structured pressure, not by multiplying vocabulary.

Every work item in this programme should therefore answer one or more of these questions:

1. Does AMS explain a stable formation more coherently than brute object-language?
2. Does AMS explain unstable or transient formations rather than merely successful stable ones?
3. Does AMS scale from T1B grouped micro-organisation to T1C higher-order bounded wholes without changing ontology?
4. Does AMS preserve one world across matter, strain, threshold, persistence, and higher-order identity?

## Scope

This programme begins in three linked workstreams:

1. `T1B Stable Molecular Set`
2. `Non-Persistent Vorton Topology Atlas`
3. `T1C Formation Stress Tests`

These are sequenced. They should not be treated as three unrelated databases.

## Ontological Anchors

The programme assumes the current AMS technical grammar:

- `T0`: substrate capability / possibility space
- `T1A`: primitive geometry, topology, adjacency, elemental relation
- `T1B`: grouped micro-organisation
- `T1C1`: bounded individual objecthood
- `T1C2`: stable kind or category
- `T1C3`: persistence and reproduction of kinds through time

Mechanism-facing AMS terms to be used consistently:

- closure
- constraint
- coupling
- lock-up
- slip
- strain distribution
- return-path
- admissibility
- metastability
- threshold
- higher-order identity

## Evidence Discipline

This research must not drift into fake precision.

Every entry should clearly separate:

- `Observed chemistry or material behaviour`
- `AMS inference`
- `Strong conjecture`
- `Open question`

This distinction is mandatory. If it is not enforced, the framework becomes rhetoric rather than research.

Every entry should also carry a `1-10 confidence score`.

That score does not mean “truth level.” It means:

- how well constrained and comparatively defensible the AMS interpretation currently is within the active corpus

The score must remain revisable as the corpus grows.

## Workstream 1: T1B Stable Molecular Set

### Aim

Create a representative `T1B` corpus of stable molecular or grouped micro-organisational formations across the periodic table.

The goal is not to chase exhaustive coverage first. The goal is to identify chemically and structurally informative cases.

### Selection Rule

Do not choose the “top 10 molecules” for each element by popularity alone.

Instead, for each element choose `up to 10` representative T1B formations that maximise structural diversity across:

- diatomic
- simple covalent
- ionic
- network
- coordination / complex
- metallic micro-organisation where relevant
- allotrope-linked cases where relevant

### Required Entry Fields

Each T1B entry should contain:

- `Element`
- `Formation name`
- `Conventional formula`
- `Conventional structure type`
- `Bonding mode`
- `Geometry / symmetry`
- `Persistence conditions`
- `Phase tendencies`
- `Known reactivity profile`
- `Known threshold or instability notes`
- `Observed explanatory challenges`
- `AMS vorton relation grammar`
- `Confidence rating`

### AMS Vorton Relation Grammar

For each formation, the AMS field should describe:

- proposed closure type
- coupling type
- lock-up conditions
- slip tolerance
- strain distribution
- return-path viability
- threshold fragility or robustness
- whether the formation behaves more like:
  - simple grouped closure
  - nested closure
  - distributed closure
  - resonant alignment
  - metastable bounded regime

### Output Standard

A T1B entry should not merely rename conventional chemistry.

It should test whether AMS can say something coherent about:

- why the formation persists
- why it holds this geometry
- why it fails under particular conditions
- how local relation-patterns produce the whole

## Workstream 2: Non-Persistent Vorton Topology Atlas

### Aim

Catalogue non-persistent or transient topology classes.

AMS should not only explain what survives. It must also explain why many closure-patterns fail, dissipate, oscillate, collapse, or remain metastable only.

### Core Taxonomy

Initial topology classes should include:

- `Diffuse`
- `Collapse-prone`
- `Overconstrained`
- `Underreturned`
- `Unstable oscillatory`
- `Incompatible adjacency`
- `Threshold-fragile`
- `Metastable-only`
- `Locally viable but globally incoherent`
- `Resonantly destabilised`

### Required Entry Fields

Each topology class should contain:

- `Topology class name`
- `Definition`
- `Why it is non-persistent`
- `Failure signature`
- `Expected runtime behaviour`
- `Likely threshold conditions`
- `Possible temporary persistence modes`
- `Possible interaction with stable T1B structures`
- `Potential catalytic / destabilising / scattering behaviour`
- `AMS confidence rating`

### Interaction Questions

For each non-persistent topology, ask:

- does it destabilise a stable T1B formation?
- can it transiently bind?
- can it act as a strain trigger?
- can it catalyse reconfiguration?
- does it scatter, resonate, dissipate, or lock locally before failure?
- does it contribute to defect formation, phase transition, or hysteresis?

This is the negative-space ontology of the framework.

## Workstream 3: T1C Formation Stress Tests

### Aim

Test whether AMS scales from T1B grouped formations into `T1C` higher-order bounded wholes.

The question is not only “can molecules exist?” The deeper question is:

- when do grouped structures become genuine higher-order wholes rather than mere aggregates?

### Initial T1C Candidate Set

The first T1C stress-test set should include:

- crystals
- glasses
- polymers
- membranes
- alloys
- granular assemblies
- catalytic surfaces
- biological macromolecular assemblies
- phase-regime objects
- defect-bearing solids

### Required Entry Fields

Each T1C entry should contain:

- `Formation type`
- `Constituent T1B basis`
- `Boundary conditions`
- `Persistence profile`
- `Failure modes`
- `Memory behaviour`
- `Threshold behaviour`
- `Whether the whole carries properties not reducible to one constituent alone`
- `AMS higher-order identity assessment`
- `Transition status`
  - aggregate only
  - marginal whole
  - stable higher-order whole
  - kind-bearing whole

### T1C Stress-Test Question Set

Each entry should answer:

1. Is the whole merely many constituents together?
2. Does disturbance redistribute through the whole?
3. Does the whole exhibit retention-properties of its own?
4. Does the boundary matter to its identity?
5. Does it show memory, preferred behaviour, or threshold response?
6. Does the whole remain meaningful across part replacement?

If the answer to these questions stays weak, the case may remain T1B aggregation rather than true T1C formation.

## Pilot Phase

### Why A Pilot Is Necessary

Do not start with all elements at once.

The framework must first prove that it can carry a spread of very different chemical behaviours without becoming ungovernable.

### Recommended Pilot Elements

The pilot element set should be:

- H
- C
- N
- O
- Na
- Mg
- Si
- P
- S
- Fe

This gives:

- simple bonding
- polarity
- ionic behaviour
- network behaviour
- organic centrality
- coordination and metallic complexity

### Pilot Deliverables

The pilot phase should produce:

1. `Pilot T1B corpus`
2. `Initial non-persistent topology atlas`
3. `T1B to topology interaction notes`
4. `Initial T1C stress-test set`
5. `Research problems list`

## Comparison Discipline

The framework should compare AMS against at least two weaker interpretive habits:

1. `Brute object-first shorthand`
2. `Description-only formalism`

For each major case ask:

- does AMS reduce explanatory fragmentation?
- does AMS preserve one ontology across cases?
- does AMS explain persistence and failure together?
- does AMS scale more coherently into T1C?

## What Counts As Success

Success does not mean proving AMS in a final deductive sense.

Success means showing that:

- AMS produces a more coherent explanatory map
- it handles stable and unstable structures together
- it reduces later ad hoc patching
- it scales from T1B to T1C without ontological rupture

The confidence scores should rise or fall as that comparison becomes more demanding. Early scores are provisional by design.

## What Counts As Failure

The programme should count failure honestly where:

- AMS adds labels without adding explanatory power
- AMS cannot distinguish persistence from instability in a disciplined way
- T1C escalation becomes arbitrary
- the same behaviour requires several incompatible AMS explanations

It should also count as methodological failure if confidence scores are inflated early and then become useless for later comparison.

That is the technical path forward for AMS.
