# AMS Glass-to-Crystal Compatibility Study v1

## Purpose

This document studies the current highest-value interface frontier:

- `glass -> crystal`

The reason this pairing matters is that both wholes are transmissive, but they appear to transmit through different organisational logics:

- glass through distributed permissive order
- crystal through structured nested order

So the main question is not:

- why does one transmit and the other reflect?

It is:

- how can one transmissive grammar hand off into another transmissive grammar?

This makes `glass -> crystal` the strongest current test of whether the interface model can carry cross-whole compatibility without collapsing into vague language.

## Current Variables in Scope

Primitive variables:

- `B_in`
- `B_out`
- `J_surf`
- `Q_coh`
- `D`

Supporting bulk variables:

- `R`
- `S`
- `J_dist`

Derived term under pressure:

- `I_comp`

## Current Working Picture

### Glass Side

Glass currently appears best described as:

- a real whole
- disordered at fine scale
- transmissive by distributed admissible pathways rather than rigid periodic order

Dominant tendencies:

- medium/high `R`
- high `S`
- medium `J_dist`
- medium/high `D`

### Crystal Side

Crystal currently appears best described as:

- a real whole
- more strongly ordered at fine scale
- transmissive by structured nested admissibility

Dominant tendencies:

- high `R`
- lower but more patterned `S`
- high `J_dist`
- low `D`
- high `Q_coh`

## Why the Interface Is Hard

This interface is difficult because neither side is simply “bad” at transmission.

The difficulty lies in the handoff itself:

- glass can carry propagation through distributed permissive reseating
- crystal appears to prefer more strongly patterned onward pathways

So the interface must do more than:

- release from glass
- admit into crystal

It must mediate a transition between two different transmissive organisations.

This is why `glass -> crystal` is currently the strongest case for `I_comp`.

## Core Compatibility Hypothesis

The present working hypothesis is:

- successful glass -> crystal handoff requires the interface to compress distributed permissive propagation into a more selectively ordered admissibility pattern without destroying coherence

That suggests three interface burdens at once:

1. preserve enough coherence from the glass side
2. prevent distributed propagation from collapsing into scattering at the boundary
3. present the propagation to the crystal in a form the crystal can admit into patterned order

If that is right, then the interface is not passive. It is a transition geometry.

## Interface Scenarios

## Scenario 1: Clean Compatibility Zone

### Description

The glass-side surface is relatively clean and the crystal-side boundary is well ordered. The interface region provides a stable transition layer rather than a hard mismatch.

### Expected Outcome

- selective successful handoff

### Best Current Variable Profile

- `B_out`: high
- `B_in`: medium/high
- `J_surf`: high
- `Q_coh`: high
- `D`: low/medium
- `I_comp`: high

### Interpretation

This is the optimistic case.

The interface layer functions as a compatibility zone rather than as a mere boundary. It allows:

- distributed release from the glass side
- narrowing into more structured admissibility
- onward propagation within the crystal

### Significance

If this case is viable, then `I_comp` is not decorative. It marks something real about interface mediation between distinct transmissive orders.

## Scenario 2: Partial Compatibility with Boundary Loss

### Description

The interface permits some distributed-to-structured reseating, but not strongly enough for dominant handoff.

### Expected Outcome

- partial handoff
- some scattering
- some structured reflection or local redirection

### Best Current Variable Profile

- `B_out`: medium/high
- `B_in`: medium
- `J_surf`: high
- `Q_coh`: medium
- `D`: medium
- `I_comp`: medium

### Interpretation

This is probably the default non-ideal case.

The interface does some real mediation, but not enough to transform the glass-side propagation fully into crystal-side admissibility.

### Significance

This case supports treating `I_comp` as graded. It is not all-or-nothing.

## Scenario 3: Boundary Collapse into Scattering

### Description

The interface cannot preserve distributed glass-side propagation well enough to present a coherent, admissible event to the crystal.

### Expected Outcome

- dominant scattering

### Best Current Variable Profile

- `B_out`: medium
- `B_in`: low/medium
- `J_surf`: low/medium
- `Q_coh`: low/medium
- `D`: high
- `I_comp`: low

### Interpretation

The propagation reaches the interface, but the transition fails.

The crystal does not admit the event into structured order, and the interface lacks enough coherent mediation to preserve a strong alternative outcome such as clean reflection.

### Significance

This case shows that two transmissive wholes do not automatically form a good transmissive interface.

## Scenario 4: Boundary Retention or Trapping

### Description

The interface neither hands off cleanly nor scatters the disturbance immediately. Instead, the event is repeatedly reseated or retained near the boundary.

### Expected Outcome

- dominant trapping
- or prolonged partial handoff with local retention

### Best Current Variable Profile

- `B_out`: low/medium
- `B_in`: low/medium
- `J_surf`: medium/high
- `Q_coh`: medium
- `D`: medium/high
- `I_comp`: low/medium

### Interpretation

This is the most underdefined case, but it matters because it would mean:

- the interface is active
- the interface is not incoherent enough for pure scattering
- the interface is not compatible enough for clean transfer

### Significance

If this case proves real, then `glass -> crystal` becomes one of the best reasons to build a stronger trapping geometry.

## What `I_comp` Seems To Mean Here

At this stage, `I_comp` looks strongest in this pairing when it means:

- the degree to which the interface can transform one transmissive organisation into another without dominant incoherence

That is stronger than:

- simple admission
- simple release

and more specific than:

- generic boundary quality

So the current study suggests:

- `I_comp` should survive
- but as a derived transition descriptor, not as a primitive substitute for other variables

## Candidate Geometric Features Behind `I_comp`

The current model is still too abstract unless it identifies what geometric features might actually raise or lower compatibility.

Most plausible candidates so far:

### 1. Transition Layer Thickness

A harder, sharper boundary likely makes distributed-to-structured reseating more difficult.

A finite transition layer may improve compatibility by allowing:

- staged narrowing of admissible pathways
- partial ordering before full crystal admission

### 2. Pathway Funnel Structure

The interface may need local geometries that:

- collect distributed propagation
- narrow it
- present it into patterned onward routes

This is one of the clearest places where the current model still needs stronger geometry.

### 3. Surface Defect Patterning

Not only defect amount, but defect arrangement may matter. Random surface disruption likely degrades compatibility faster than structured transition irregularity.

### 4. Coupling Gradient

The interface may need a gradient in local coupling rather than an abrupt shift from glass-style to crystal-style organisation.

This would make `J_surf` more than “strong or weak.” It would make it spatially graded.

## Current Strongest Findings

## 1. This Pairing Justifies `I_comp` Better Than Any Other Current Case

Not because `I_comp` is mathematically sharp yet, but because this is the case where:

- admission and release both matter
- both wholes are transmissive
- the main difficulty is transition between transmissive grammars

## 2. `J_surf` Is Carrying Heavy Interface Work

The current study suggests that:

- `J_surf` is one of the strongest primitive variables in this pairing

The interface appears to need active mediation, not only passive contact.

## 3. `Q_coh` Still Matters, But Less Exclusively Than in Air -> Crystal

In air -> crystal, coherence compatibility seems dominant.

In glass -> crystal, coherence still matters, but interface mediation and transition geometry seem to matter more.

## 4. `D` Is Important but Not Primary

This is not mainly a defect-dominated case unless the interface is poor.

The central issue is compatibility of organisation, not raw damage.

## Current Weak Points

## 1. No Explicit Funnel Geometry Yet

The current study points toward something like a pathway funnel or compatibility zone, but the geometry is still verbal rather than formal.

## 2. Trapping Remains Weakly Specified

This pairing may reveal real boundary retention behaviour, but the current model cannot yet describe it sharply.

## 3. `J_surf` May Need One More Split

At present `J_surf` carries:

- surface mediation strength
- degree of transition-layer coupling
- capacity for local reseating

That may be too much.

Possible future split:

- `J_med`
- `J_trans`

or some equivalent distinction between:

- immediate surface coupling
- through-interface transition coupling
