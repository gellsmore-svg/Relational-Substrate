# AMS Core Runtime Machines Comparative Synthesis v1

## Purpose

This note compares the two strongest integrated runtime machines currently mapped in the AMS programme:

- respiration plus ATP synthesis;
- ribosome plus translation.

These are not just two interesting systems. Together they test whether the core family grammar can explain:

```text
energy-to-order conversion
and
information-to-architecture conversion
```

under one ontology.

The locked family identities are:

```text
Carbon carries molecular architecture.
Nitrogen carries biological availability and recognition.
Phosphorus carries transferable biological order.
Sulfur carries controllable chemical reactivity.
Iron carries redox depth.
Oxygen carries polarity and oxidative power.
```

## The Two Machine Classes

### 1. Respiration / ATP Synthesis

Primary runtime sequence:

```text
iron depth
-> sulfur-governed relay
-> oxygen terminal closure
-> proton-field asymmetry
-> phosphorus capture as ATP
```

This is the main:

```text
redox-to-order machine
```

### 2. Ribosome / Translation

Primary runtime sequence:

```text
nitrogen-readable sequence
-> phosphorus-backed readable track
-> oxygen-conditioned aqueous legibility
-> carbon protein architecture
```

This is the main:

```text
information-to-architecture machine
```

## Core Symmetry

The most important result is that these machines are not unrelated.

They share the same higher-order structure:

```text
1. a constrained input
2. a readable or routable machine host
3. a directional cycle
4. a field-conditioned environment
5. a captured output
6. a characteristic failure boundary
```

### Respiration

```text
input = reducing equivalents
host = membrane-embedded redox machine
cycle = electron descent and proton pumping
field = oxygen-water-conditioned proton asymmetry
output = ATP-capable ordered state
failure = ROS, uncoupling, oxidative collapse
```

### Translation

```text
input = readable nucleotide sequence plus charged amino-acid packets
host = ribonucleoprotein decoding machine
cycle = codon reading, delivery, elongation, translocation
field = oxygen-conditioned aqueous readability environment
output = protein architecture
failure = mistranslation, stalled or degraded product, coherence loss
```

This shared shape is strong evidence that AMS is not merely cataloguing chemistry. It is describing runtime-machine grammar.

## Oxygen as Shared Family

Oxygen is the strongest shared family across both machines.

But its role differs sharply.

### In Respiration

```text
oxygen = terminal sink + proton-field condition
```

### In Translation

```text
oxygen = hydration and charge-conditioned readability field
```

This is a major success of the oxygen family lock. The same identity:

```text
polarity and oxidative power
```

remains coherent across both machines without collapsing into vagueness.

## Phosphorus as Shared Transfer Family

Phosphorus is also strongly shared, but again in different ways.

### In Respiration

```text
phosphorus = captured runtime packet at the end
```

### In Translation

```text
phosphorus = readable backbone continuity and directional cycle permission throughout
```

This confirms that phosphorus is not "energy only." It is the broader family of transferable biological order.

## Carbon as Asymmetric Family

Carbon appears differently in the two machines.

### In Respiration

```text
carbon = host architecture and upstream fuel context
```

### In Translation

```text
carbon = final product architecture
```

This asymmetry is useful. It shows that the same family can play either:

- host role;
- substrate role;
- product role.

without changing its locked identity.

## Nitrogen as Asymmetric Family

Nitrogen is similarly asymmetric.

### In Respiration

```text
nitrogen = protein readability and contextual embedding
```

### In Translation

```text
nitrogen = front-end readability itself
```

This strongly confirms the nitrogen lock:

```text
Nitrogen carries biological availability and recognition.
```

Translation is where that sentence is most directly machine-real.

## Sulfur and Iron as Redox-Side Specialists

Sulfur and iron are much more direct in respiration than in translation.

This is not a defect. It is an expected family asymmetry.

```text
iron = redox depth specialist
sulfur = dangerous-chemistry governance specialist
```

These families are not meant to dominate every runtime machine. They dominate the machines where deep redox routing must be made governable.

That is the strongest current paired machine statement in the programme.

- stalling;
- premature termination;
- defective product architecture;
- folding overload.

The difference is important:

```text

translation failure = information/architecture coherence loss
```
