# AMS Corpus State and Next-Phase Summary v2

## Purpose

This note refreshes the programme-control summary after the machine-calibration phase and the updated full-thesis synthesis.

The goal is to answer four questions clearly:

```text
1. What now exists?
2. What has materially improved since v1?
3. What remains weakest?
4. What should the next phase now focus on?
```

This is not a replacement for the local files. It is the current control summary for the state of the corpus.

## Central Claim

The central claim is:

> The AMS corpus has moved from a conceptually ambitious but uneven body of work into a more layered and controlled research programme with four real strengths: a largely locked element-family core, a threshold-bearing machine layer, a stronger life-to-person boundary ladder, and a more disciplined corruption-redemption arc. The next phase should now focus on selective reintegration and the few remaining unresolved calibration questions rather than broad new extension.

More briefly:

```text
The corpus is now stronger in the middle than before, and more selective next moves now matter.
```

This remains the clearest empirical-comparative centre of gravity in the programme.

## 2. Runtime-Machine Layer

The machine layer is now one of the strongest parts of the corpus.

Its two anchor systems remain:

```text
Respiration / ATP synthesis
Ribosome / translation
```

But these are now stronger than they were in `v1` because they exist not only as:

- integrated route maps;
- comparative syntheses;
- failure-propagation structures;

but also as:

- threshold-calibration files;
- machine-specific state bands;
- machine-to-cell transition logic.

This is one of the most important changes since the earlier summary.

## 3. Boundary Layer

The boundary ladder remains:

```text

-> life
-> person
```

with dedicated files for:

- living-state boundary;
- AI versus living systems;
- personhood versus living systems;
- life edge cases;
- personhood edge cases.

This remains one of the strongest conceptual gains of the wider corpus.

## 4. Cell-State Layer

This layer is now clearer than before because the machine-calibration phase has tightened the bridge from:

```text
local machine stress
```

to:

```text
whole-cell state change
```

The key gain is that the cell is no longer described loosely as a mere collection of machines, nor so abstractly that machine thresholds stop mattering.

The current cell-state grammar is now more explicitly:

- coherent state;
- stress compensation state;
- failure state.

## 5. Arc Layer

The upper ontology now has a materially stronger arc layer, including:

- corruption as runtime fragmentation;
- redemption as restorative reordering;
- restoration versus replacement;
- resurrection continuity stress-testing;
- refreshed full-thesis synthesis.

This layer is still more synthetic than the machine and family core, but it is no longer the major weak point it once was.

## 6. Formal Scaffold Layer

The programme still has a first semi-formal substrate scaffold:

- vorton geometry and closure formalism;
- route-variable framework;
- topology transition criteria;
- family-profile mapping back into the formal layer.

This remains incomplete, but the key change is still:

```text
the main problem is no longer absence of scaffold
```

but:

```text
the scaffold exists and now needs selective refinement and disciplined reuse
```

## What Has Materially Improved Since v1

The recent work has improved the corpus in five especially important ways.

## 1. The Middle Layer Is No Longer Loose

In `v1`, the machine layer was improving but still sat below its potential as a constraining layer.

That is no longer true.

The machine layer now carries:

- variable discipline;
- threshold bands;
- failure-style differentiation;
- machine-to-cell export logic.

That means the ontology is now stronger precisely where family identities, living continuity, and corruption logic need a disciplined bridge.

## 2. Machine Diversity Is Better Preserved

One of the main risks in a unified ontology is flattening everything into one type of system behaviour.

That has been reduced.

Respiration now reads primarily as:

- pressure-loaded;
- leak-sensitive;
- oxidatively fragmentable.

Translation now reads primarily as:

- fidelity-loaded;
- ambiguity-sensitive;
- burden-accumulating;
- slower to collapse but easier to drift.

This is a major gain because it shows the formal and machine language can remain discriminating rather than merely repetitive.

## 3. The Life Boundary Is Better Constrained

The life boundary is now less purely definitional because it sits on a clearer machine-to-cell bridge.

The strongest current line is no longer just:

```text
life involves self-renewing closure
```

but:

```text
life persists while recoverable machine-remaking coherence remains owned by the system strongly enough that exported burdens do not abolish whole-cell recoverability
```

That is a more disciplined ontology of living continuity.

## 4. The Top Thesis Is Better Integrated

The new `ams-full-ontology-core-thesis-synthesis-v3.md` is better than `v2` because it explicitly absorbs:

- threshold-bearing runtime machines;
- calibrated drift and collapse distinctions;
- the tighter machine-to-cell bridge.

This is important because the summary layer is no longer lagging as badly behind the strongest local work.

## 5. The Programme Is More Selective About Its Own Needs

The corpus now knows more clearly what it is missing.

The weakness is no longer:

```text
we need more categories
```

It is now closer to:

```text
we need selective reintegration, unresolved-score review, and careful technical deepening
```

That is genuine progress.

## What Remains Weakest

Even after the machine-calibration phase, several weaknesses remain identifiable.

## 1. The Formal Scaffold Is Still Only Semi-Formal

The scaffold now exists and is more useful than before, but it still remains:

- comparative;
- band-based;
- lightly mathematical rather than deeply formalised;
- not yet anchored in a weighted multi-system framework.

That is acceptable for now, but it remains a limit.

## 2. Cross-Machine Weighting Is Still Broad

The machine layer is materially stronger, but:

- export channels are not yet explicitly weighted;
- cascade timing remains broad;
- machine-to-cell propagation is still comparative rather than rigorously quantified.

This is the clearest technical deepening path if the next phase continues downward.

## 3. Iron Remains the Main Unresolved Family Question

The element-family core is strong, but one score decision remains meaningfully open:

```text
whether iron should remain at 9.25/10 provisional upward revision or now be more firmly locked
```

That is the clearest unresolved comparative-review question in the current corpus.

## 4. Outward Extension Still Carries High Risk

The programme is now better prepared for:

- social;
- ecclesial;
- civilisational;

applications, but these remain the highest-risk zone for premature extension.

They should still be delayed until either:

- reintegration is complete;
- or technical deepening is stronger.

## Next-Phase Priorities

The next phase should focus on three things.

## 1. Controlled Reintegration

Highest-value targets:

- the refreshed full-thesis file;
- the refreshed corpus summary layer;
- any living-state summary language that still lags behind the stronger machine bridge.

This should happen before more broad extension.

## 2. Iron Review

The best remaining family-level review target is:

```text
iron final lock review
```

because the family is strong enough that the remaining question is no longer identity but final confidence discipline.

## 3. Selective Technical Deepening

If work continues downward rather than upward, the best targets are:

- cross-machine export weighting;
- stronger machine-to-cell burden coupling;
- one more comparative calibration pass if it materially increases discrimination.

This is a better next move than opening major new domains.

## What Not to Do Next

The corpus is now large enough that weak next moves matter.

Avoid prioritising:

- broad new element families before current reintegration is settled;
- social or civilisational extrapolation too early;
- new high-level synthesis that adds no new constraint;
- false precision in the formal layer;
- multiplying local files without phase-control summaries.
